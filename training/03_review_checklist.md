# 交易复盘检查清单

> **状态：Training / Derived**
>
> 本清单用于训练和复盘，不新增执行规则；计划、订单、成交和管理事实以当前[执行手册](../EXECUTION_MANUAL.md)的[核心交易计划](../EXECUTION_MANUAL.md#4-一份核心交易计划)为准。

## 使用方式

每次只复盘一段走势或一笔交易。先锁定当时可见的信息，再查看后续结果；若使用逐根回放，先完成[逐根观察模板](02_bar_by_bar_templates.md)。本清单只引用原始观察和执行记录，不重抄交易计划。

## 九个核心问题

1. 当时记录是否完整保留，且没有用后续走势覆盖原判断或原计划？
2. 当时对市场结构、控制权、位置和第一现实目标空间的判断是什么；最重要的证据与反方理由分别是什么？
3. 是否只有一个清晰的交易命题和方向；若没有，等待是否是正确结果？
4. 计划使用的可观察触发条件、命题失效事实、保护性止损、仓位、目标、时间条件和管理事件是否在下单前属于同一个交易想法？
5. 概率、最差入场、最差止损成交、目标、数量和成本是否来自同一计划，并在保守假设下仍值得承担风险？
6. “允许下单”、经纪商活动订单和真实成交是否被正确区分；条件改变时是否先撤销旧订单，再建立新计划？
7. 每次真实成交后，实际数量是否立即获得完整保护；部分成交时，真实持仓与未成交余单是否分别处理？
8. 持仓后是否区分正常波动、原计划事件和新出现的 premise 失效证据；无论出现什么新信息，都没有临场放宽风险、增加数量或改写交易命题？
9. 流程最早在哪一步结束或偏离；哪一条当时记录能直接证明这个判断？

## 最终复盘输出

### 1. 流程结果

只选择一个最符合当时过程的结果，不按最终盈亏归类：

- **合格观察训练**：本轮只训练预定的市场状态、位置、证据或交易想法问题，记录完整，未进入计划或订单范围。
- **合格执行**：交易想法、计划、数学、订单和成交后管理均符合原记录。
- **合格但未成交**：计划与订单合格，最终没有真实成交。
- **合格不交易**：本轮已进入计划设计或风险筛选，但位置、空间、触发、计划或数学不合格，因此正确等待；纯观察轮不使用此分类。
- **计划未完成**：本轮已经进入计划设计或准备承担风险，却只有方向、形态或入场想法，没有形成完整计划。
- **执行偏差**：原计划可以复核，但订单、仓位、保护、目标或管理没有按其执行。

```text
流程结果：
支持该归类的当时记录：
流程结束或首次偏离的位置：
```

### 2. 市场结果

市场结果只记录后来发生了什么，用于核对假设和实际成本；它不能反向修改流程结果。

纯观察轮没有执行记录时，订单、成交和盈亏字段写“不适用”，只记录后来市场如何发展。

```text
是否真实成交 / 最终实际数量：
触发后获得跟进、失败、回到公平区域，还是形成持续反向运动：
原计划中的目标、失效、跟踪止损或时间条件后来如何发展：
实际退出原因和成交结果（如有）：
毛账户盈亏 / 实际成本 / 净结果（如有）：
```

零成交或合格不交易的样本也记录后续市场路径，但“后来走对了”不能证明当时可以跳过计划或数学。

### 3. 主要问题

一次只选择一个最值得修正的问题。问题必须引用当时记录并落到具体步骤，不使用“感觉不好”“不够耐心”等无法复核的总结。

```text
主要问题（若无流程问题，写最需要继续验证的不确定点）：
所在步骤：观察记录 / 市场判断 / 交易想法 / 完整计划 / 入场数学 / 订单与成交 / 持仓管理
当时证据：
原要求：
实际偏差：
```

```text
要复习的文件或概念：
下一段要练的场景：
这次只训练的问题：
用什么当时记录证明已有改善：
```

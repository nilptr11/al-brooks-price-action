# 连续回放训练

> **状态：Training / Derived**
>
> 本目录只组织观察、回放和复盘；价格行为判断回到相应学习内容与来源，仓库订单和记录操作见[执行手册](../EXECUTION_MANUAL.md#7-仓库订单与记录操作)，批次配置和单笔交易使用当前[执行记录模板](../EXECUTION_RECORD_TEMPLATE.md)。

## 训练目标

训练不是给后续走势贴上更多名称，而是在未知未来的连续市场过程中，留下足够清楚的当时记录，使另一位读者能够判断：你看到了什么、判断为何改变、计划是否完整，以及实际执行是否遵守原计划。

每次只选择一个**流程复盘重点**，例如“能否区分正常回调和 premise 变化”。它不是要求市场出现某个形态或问题；本轮没有遇到对应事件也不主动搜索。没有出现准备交易的 setup、计划未完成、主动放弃或订单最终零成交，也都是有效样本。

## 训练单位

训练单位是一段预先不知道结果的连续市场过程，不是一周、固定交易数量或指定形态样本。默认从上一次保存的历史位置继续推进；随机起点只在已经具备基本识图能力后用于检验迁移，不用于搜索 wedge、H2、突破或其他目标场景。

默认在同一轮一边逐根观察，一边在自然出现准备交易的 setup 时记录和模拟交易。若事先明确进行纯观察练习，可以不进入计划或订单，但该样本只检验观察能力；是否实际提交模拟订单取决于当时 context、setup、stop、target、Trader's Equation 和个人能力，而不是交易数量配额。

训练开始前先建立或引用一份[批次执行配置](../EXECUTION_RECORD_TEMPLATE.md#批次执行配置)，按执行手册固定数据源、品种、决策周期、图表、tick、风险边界和成本处理规则。手续费、模拟滑点和资金费规则由配置统一管理，不在每笔交易中重复手填。若以 5 分钟为决策周期，市场判断、entry、stop 和管理以 5 分钟结构为主；高周期只在具体价位确实影响位置、障碍或目标时作为附加参考。

## 一次完整训练

1. 引用一份有效的[批次执行配置](../EXECUTION_RECORD_TEMPLATE.md#批次执行配置)，从上次保存的历史位置开始，隐藏尚未发生的 K 线，并在开始前写明本轮流程复盘重点和结束条件。结束条件使用连续市场窗口或认知负荷，不使用交易数量或场景数量。
2. 使用[逐根观察模板](02_bar_by_bar_templates.md)记录新增事实、判断变化和下一步待验证信息；过去的记录不回写。
3. 准备交易一个 setup 时，建立一份[单笔交易记录](../EXECUTION_RECORD_TEMPLATE.md#单笔交易记录)，保存当时的 context、setup、entry、price-action stop、profit target、Trader's Equation、仓位和管理方式。
4. 若当时判断不值得交易，记录原因后继续逐根回放；若决定交易，则按记录的订单方式进行回放或模拟成交。任何真实订单仍须先满足用户自己的风险边界。
5. 订单、成交、protective-stop 覆盖和重要管理事件只追加在同一交易记录中；逐根观察表只引用该记录和时间，不复制价格、数量或动作。
6. 到达预定结束条件时保存历史结束位置，使用[复盘检查清单](03_review_checklist.md)分别记录流程结果和市场结果，只选择一个主要问题。
7. 把主要问题转换成下一轮的一个流程复盘重点；下一轮从保存位置继续，不为验证问题主动寻找场景。

盈利不能补足缺失的观察、计划或执行记录，亏损也不能单独证明当时有依据的判断错误。复盘始终以当时可见的信息为准。

## 记录边界

- 逐根模板只记录观察，不记录计划参数、订单数量、成交或管理动作。
- 交易计划、入场数学、订单、成交和管理事件只保存在执行手册规定、由当前模板生成的同一交易记录中。
- 复盘清单引用原始观察与执行记录，不再重抄一份计划。
- 后续信息只能用于记录市场结果，不能覆盖当时判断。
- Market cycle、premise、执行尺度、setup、gap 或 measured move 等索引只从当时记录被动生成；复盘发现当时分类证据不足时记录问题，不用后见之明改写标签。
- 已经看过结果的片段可以用于针对性纠错，但属于重复练习，不再作为未知未来的盲测样本。
- 需要改变仓库订单或记录操作时修改执行手册；价格行为内容有误时先修改其来源所有者，训练材料只同步引用和练习问题。

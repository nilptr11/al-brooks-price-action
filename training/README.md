# 场景训练

> **状态：Training / Derived**
>
> 本目录只组织观察、回放和复盘；规范字段与交易动作以当前[执行手册](../EXECUTION_MANUAL.md)为准。

## 训练目标

训练不是给后续走势贴上更多名称，而是留下足够清楚的当时记录，使另一位读者能够判断：你看到了什么、判断为何改变、计划是否完整，以及实际执行是否遵守原计划。

每次只练一个问题。没有形成清晰交易想法、计划未完成、数学不合格或订单最终零成交，也都是有效样本。

## 一次完整训练

1. 选择一段走势和一个训练问题，隐藏尚未发生的 K 线。
2. 使用[逐根观察模板](02_bar_by_bar_templates.md)记录新增事实、判断变化和下一步待验证信息；过去的记录不回写。
3. 只有本轮问题包含计划设计时，才在出现值得设计的交易想法后另行填写执行手册的[核心交易计划](../EXECUTION_MANUAL.md#4-一份核心交易计划)。若本轮只练市场状态、位置或证据，按预定问题完成观察即可，不必把每个候选升级成计划。
4. 只有本轮包含订单或持仓管理时，才继续练订单状态、成交保护和管理。回放或模拟成交必须明确标注为假设；任何真实订单仍须先满足执行手册使用边界，真实成交和管理事件只追加在同一执行记录中。
5. 本段结束后，使用[复盘检查清单](03_review_checklist.md)分别记录流程结果和市场结果，只选择一个主要问题。
6. 把主要问题转换成下一项学习或同类场景练习，再开始下一轮。

盈利不能把不完整流程改写为合格，亏损也不能把合格判断和执行改写为错误。复盘始终以当时可见的信息为准。

## 记录边界

- 逐根模板只记录观察，不记录计划参数、订单数量、真实成交或管理动作。
- 交易计划、入场数学、订单、成交和管理事件只保存在执行手册规定的同一执行记录中。
- 复盘清单引用原始观察与执行记录，不再重抄一份计划。
- 后续信息只能用于记录市场结果，不能覆盖当时判断。
- 需要改变执行规则时修改执行手册；训练材料只随之更新引用和练习问题。

# 形态只是语言

> **状态：Learning / Non-normative**
>
> 用于概念学习；文档职责与执行权威入口见 [`README.md`](../../README.md#权威层级)。

## 它解决什么问题

Pattern 用来命名市场行为，帮助交易者沟通和组织信息。孤立 pattern 不是交易信号；只有把它放回 context、并作为放置入场单的依据时，才达到官方 glossary 对 setup 的最低定义。

## Brooks 语境中的原则

每个形态都要回到：

- Market cycle。
- Context。
- Location。
- Order flow。
- Follow-through。
- Trapped traders。
- Target。
- Management。

同一个形态在不同环境中含义可能相反。强趋势中的 wedge pullback 可能是顺势机会；成熟通道后的 wedge 可能是反转尝试；区间中部的 wedge 可能没有优势。

## 本目录怎么读

本目录只定义形态语言。真正的交易构造见 [`05_setups/`](../05_setups/README.md)，交易管理见 [`06_trade_management/`](../06_trade_management/README.md)。

如果某个形态还没有 context 和入场依据，它只是 pattern；即使已经成为 Brooks setup，若仍无法回答“在哪里错、目标在哪里、怎么管理”，它也不是本仓库可批准的交易计划。

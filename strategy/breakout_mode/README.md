# 双向等待策略

> **定位：供交易者在方向尚未决定、结构允许双向计划时选择策略**

## 当前只有一项完整策略

本目录目前只包含[严格 ii 结构中的双向突破等待](ii_bidirectional_breakout.md)。它要求母线后连续两根严格嵌套的内包 K 线，并且两个方向都能使用同一套风险逻辑。

ioi、oo、普通三角形和一般紧区间虽然也属于 breakout mode 语言，但它们的结构、波动和订单风险不同。目前不与 ii 共用同一计划：

- 真正突破后，根据突破强度、跟进或第一次回测进入[突破延续家族](../breakout/README.md)；
- 首次突破失败并回到成熟区间时，重新检查[交易区间策略](../range/README.md)；
- 没有完整策略条件时继续等待。

## 使用双向计划的最低要求

- 形成结构时仍无法合理预判方向；
- 上下两个方向都有现实目标空间；
- 两边使用同样清楚的入场、止损、目标和数量；
- 平台能够返回单一组合状态，同时确认两张订单有效且二选一联动已经建立；分别确认两边不够；
- 已计算两边意外同时成交时的最坏风险。

如果只有一个方向合格，就不能把双向策略改成主观单向预测。

ii 页面用完整母线高度向两侧投射，是本仓库为了让两个方向在结构完成时都能固定价格而采用的 measured-move 基线；Brooks 的 breakout mode 概念本身不为所有 ii 规定这一唯一目标。任一方向若在该基线目标前存在不可合理穿越的强障碍，或扣除成本后不合格，就不建立双向计划。

相关概念见[突破与 Breakout Mode](../../core/01_market_cycle/03_breakouts_and_breakout_mode.md)和[Triangles、ii、ioi 和 oo](../../core/04_patterns/06_triangles_ii_ioi_oo.md)。

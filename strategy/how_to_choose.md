# 如何选择并锁定一项策略

> **定位：供交易者从连续观察走到一份明确计划**

本页是 `strategy/` 中全局生命周期、时段覆盖、单候选约束、计划锁定、结束与重入的唯一权威状态机。目录页和策略地图只摘要并链接这里；状态机路由到某个家族后，由该家族 README 负责家族内分类与重叠优先级，具体策略页负责该 setup 的资格、触发、参数和特有失效。家族规则不得反向改写本页的全局顺序或时段覆盖。

价格、风险、时间与管理字段的共同含义来自[策略共同规则](common_rules.md)，具体形成条件来自所选页面。本页不规定订单提交、回执、成交、实际仓位或异常处置；[执行手册](../execution/execution_manual.md)只消费已经锁定的策略计划并把它安全落地，不参与策略选择。

## 唯一选择与覆盖状态机

一轮执行从连续观察开始，不从寻找某个形态开始。状态只能沿以下顺序前进：

1. **定义观察边界**：先写定执行环境、品种、交易周期、外层周期、session、观察结束时间、账户风险边界、限制 Personal risk 的纪律护栏、计划持仓期限，以及已知日程事件、基础设施与尾部风险边界。
2. **形成当前 Context**：只用当时可见的信息判断 market cycle、重要位置、最近主导控制事件、当前 Always In 方向及反证、前方 magnet 和路径障碍。状态不清时只使用下一节的逐层 fallback。
3. **检查时段覆盖**：判断是否进入开盘或尾盘窗口，以及具体时段页的候选起点是否已经出现。同一底层机会被时段页覆盖时，只能进入该时段候选；没有重叠时才继续检查普通家族。
4. **路由到一个价格行为家族**：根据当前 Context 进入趋势、突破、区间、双向等待或反转家族；时段始终是上一步的覆盖层，不与这些家族并列。形态语言、Always In、Pain Trade 或 fallback 标签都不能单独完成这一步。
5. **在家族内保留一项候选**：按该家族 README 的分类与重叠规则，再对照具体页的适用环境、形成顺序和不适用环境，只保留一项活动候选。条件不完整就继续观察；冲突无法消除就等待或放弃。
6. **通过共同门槛**：具体页条件完整后，同时检查结构止损、现实目标、成本、四类风险、Trader's Equation、实际可用的剩余持有时间、`latest_entry_time`、`forced_exit_start`，以及日程事件、基础设施可用性和尾部风险的共同门槛。
7. **锁定一份计划**：下单前一次写定本页要求的全部字段，随后只追加新事实。承担风险前若 entry、stop、target、数量、时限或管理分支需要实质改变，必须结束旧计划并从当前价格重新判断；成交后只能执行预写分支，超出分支的新变化不能覆盖原值，应先按原计划安全结束风险，再把后来机会建立为新计划。
8. **结束或重新进入**：候选被拒绝、失效、过期、未成交，或计划成交后归零，都会结束旧状态。后续机会必须包含旧计划结束后才完成的新当前结构或触发，并重新走完以上步骤；旧名称、信号、触发和订单不能复用。

任何时刻最多只有一项活动候选或计划。强突破收盘、跟进和第一次回测等相邻进入时点可以先后出现，但不是自动建立多份风险的理由。

### 状态不清时的逐层 fallback

选择状态机继承[市场周期的逐层 fallback](../core/01_market_cycle/00_market_cycle.md#状态不清时的逐层-fallback)，且每次只处理当前未决层级：

- 连 trend 与 trading range 都分不清时，暂按 trading range 管理；
- 已确认趋势、但分不清 breakout phase 与 channel 时，暂按 breakout phase 管理；
- 已确认 channel、但分不清 tight 与 broad 时，暂按 tight channel 管理；
- 分不清 broad channel 与 trading range 时，暂按 trading range 管理。

Fallback 只是信息不足时用来限制风险和避免采用过度激进逻辑的保守管理假设，不是市场真实标签的确认，也不产生策略许可。“暂按 trading range 管理”不等于已经形成成熟区间、价格位于可交易边缘或允许逆势 limit / scale-in；“暂按 breakout phase / tight channel 管理”也不等于已经满足任一突破或趋势策略。只有具体策略页的 Context、位置、形成顺序、触发和共同门槛独立完整，状态机才可进入候选或计划。新证据出现后必须重新分类。

### 阶段更新不是新增策略

选择器必须随当前价格更新趋势阶段，而不是把最初标签永久保留：

- 强 breakout phase 在选定周期出现第一个可识别 pullback 后，该段 breakout phase 已结束，状态更新为 channel；整个 trend 不因此自动结束，后续也可能形成新的 breakout phase。
- Pullback 覆盖前一突破点、相邻突破与回调开始重叠时，staircase 表明开放空间关闭、逆势方开始有获利机会，应降低强趋势假设并重新判断 channel / trading range。Shrinking stairs 进一步增加过渡证据，但两者都不是反转触发。
- Broad channel 与 trending trading range 可以是同一段价格的两种职责视角，不能作为两份独立证据重复计票。前者强调方向性高低点和深回调，后者强调局部平衡区与短突破；选择器应按当前计划所依赖的结构重分类。

这些都是 Context 与阶段路由信息，不增加独立策略，也不能让同一次价格运动重复生成候选。具体边界见[趋势和通道](../core/01_market_cycle/01_trends_and_channels.md#breakout-首次回调staircase-与阶段重置)。

## 时段策略怎样覆盖普通策略

开盘与尾盘不是趋势、区间或突破之外的另一种市场状态，而是加在当前市场状态上的时间覆盖层。进入预先写定的时段窗口后，先检查[开盘与尾盘策略](session/README.md)。这里的覆盖对象是正在活动的同一底层交易机会：它既不狭窄到只认同一根信号，也不会因一次触碰就永久锁定整段后续行情。

- 具体时段页规定的候选起点出现后，该时段候选开始活动；只在时段窗口内、但尚未达到任何页面候选起点，不会覆盖无关的普通机会。
- 活动时段候选与普通页拟议入场依赖同一参照事件、预先边界和方向时，使用时段页。即使普通页等待较早或较晚的信号、采用不同入场时点，也仍是同一底层机会；不能借此省略确认、绕过六十或九十分钟截止、替换目标，或逃避尾盘强制退出。
- 同一时刻只保留一项活动候选或计划。若同一底层机会同时符合两张时段页，而当时信息不足以唯一说明主要交易前提，就继续等待或放弃。
- 活动候选或计划结束前，不建立第二份风险。候选被拒绝、失效、过期或未成交后，原信号、原入场依据和原订单不能换名或复用。
- 候选或计划结束后，从当前价格重新判断。新计划必须包含旧计划结束后才完成、且未被旧计划使用的当前结构或触发，重新满足所选策略全部条件，并重新计算入场、止损、目标、数量和剩余时间。已经完成的突破、接受、趋势或区间结构可以作为新策略明确要求的历史背景，但不能直接替代新的当前信号。
- 时段页条件不完整、太晚或过期，不会自动授权普通页；若普通页拟议入场仍依赖原时段候选的参照事件、边界和方向，而且旧候选结束后没有形成新结构或新触发，它仍是旧机会，必须放弃。相反，后来真正形成的新当前机会可以重新评估，不受一次更早触碰永久锁定。
- 当前机会与时段页并不重叠时，普通策略可以独立使用；仍须服从本轮预先写定的观察结束时间、剩余时段和风险边界。

例如，开盘区间突破计划未成交过期，而且此前从未回测开盘区间边界；其后第一次触及边界形成新的当前参照事件，可以重新检查突破后的第一次回测。原突破和接受 K 线只提供该新策略要求的历史背景；首次触及、恢复信号、入场、止损和目标必须由新计划重新确定。若边界在旧计划结束前已经被触及，后来再次触及就不是“第一次回测”，不能借此复用旧机会。

## 什么情况下开始记录一个机会

只有当某个策略页规定的最早可观察条件已经出现，才把它记为候选。记录：

- 当前市场环境和位置；
- 这项策略为什么可能成立；
- 哪些证据支持，哪些证据反对；
- Always In 所属周期、最近主导的来源事件、当前方向和尚未推翻它的理由；
- 可能推翻或已经削弱当前 Always In 判断的反向证据；
- 下一项必须出现的确认；
- 什么事实出现就立即放弃。

候选不等于交易。后来被拒绝、过期、没有触发或没有成交的候选仍然保留。

Always In 的来源事件应落到可定位的突破、spike、连续压力或控制转移，而不是只写 `AIL` / `AIS`。当前方向必须写为 `Always In Long`、`Always In Short` 或 `unclear`，并保留反证；它是当前周期的强控制判断，不是实际仓位，也不是无条件入场许可。实际是否持仓、方向和数量是执行事实，按[执行手册](../execution/execution_manual.md#八成交以后先更新仓位)另行核对。

## 什么情况下条件完整

条件完整以前，必须能够清楚回答：

1. 当前背景为什么支持这项策略？
2. 机会的必要结构是否按时间顺序完整出现？
3. 入场触发具体证明什么？
4. 哪个价格事实会否定原交易理由？
5. 保护止损是否位于结构外，而不是正常波动中？
6. 目标区域是否来自当前结构或可解释的 magnet，页面基线目标和实际退出价格是否已按共同口径唯一确定？
7. 事先可见的强障碍是否已经写出具体价格和来源？
8. 数量是否已经按结构止损距离和账户风险边界确定，最坏亏损是否可承受？
9. 按[共同价格与净 R 口径](common_rules.md)扣除费用和保守滑点后，净风险、净回报和净 R 分别是多少？
10. 把当前概率依据、止损、现实回报、前方障碍和管理方式放在一起，Trader's Equation 是否仍然有利？
11. Initial / price risk 与计划 account risk 是否分别成立；账户亏损上限、仓位、不得扩大止损或临时加仓等事前护栏是否足以限制 Personal risk；是否避免借用事后 Actual Risk / MAE 改善计划？
12. 当前实际可用的剩余持有时间是否达到 `minimum_remaining_holding_time`，足以容纳这项策略的正常波动、目标路径与管理？
13. 当前是否严格早于 `latest_entry_time`，且未成交余量能在该时点结束？
14. `forced_exit_start` 及退出缓冲是否已经明确，并能由当前执行环境安全实现？
15. 已知日程事件、基础设施异常与尾部风险是否已进入计划；当前环境能否持续核对仓位、维持保护并安全退出？
16. 同一次机会是否已有另一项活动计划？

任何必要事实看不清，都只能继续等待或放弃。

## 下单前一次写定什么

- 所选策略、方向、premise、交易周期和管理周期；
- Always In 的来源事件、所属周期、当前方向、支持证据、反证和更新条件；
- 如适用，记录 `signal_bar` 的时间与角色；仍在形成时明确写作 `prospective signal bar`。没有 signal bar 的策略记录其页面规定的其他参照事件；
- `chart_entry_trigger`、订单方式和计划价格；`chart_entry_bar` 只在图表第一次满足触发后填写；
- 未成交时的撤单和过期条件；
- 保护止损的位置和理由；
- 目标区域及来源、页面基线目标、实际退出价格和持仓管理；
- 目标区域前最近强障碍的价格和来源；
- 基础数量、是否允许预先计划的加仓，以及最坏情况下的预计亏损；
- Initial / price risk、计划 account risk，以及限制 Personal risk 的账户亏损上限、仓位、止损与纪律护栏；
- 最低剩余持有时间、`latest_entry_time`、`forced_exit_start` 和对应退出缓冲；
- 到达 `latest_entry_time` 后撤单竞态产生晚到成交时的预写分支；
- 已知日程事件的风险窗口与禁止新风险／强制退出边界，基础设施主备路径，以及跳空、流动性、停牌／涨跌停、隔夜或结算等适用尾部风险；
- 按保守成交与实际退出价格计算的计划净风险、计划净回报和计划净 R；
- Trader's Equation 的判断及其概率依据；
- 交易理由削弱或失效时怎样退出。

Signal bar 与 chart entry bar 的语义见[Signal Bar 与 Entry Bar](../core/04_patterns/01_bar_types.md#signal-bar)。`actual_fill_bar` 只属于账户真实成交事实，回放／模拟则使用所选环境确认的 `simulated_fill_bar`；二者都不属于锁定策略计划的输入，也不能回填或替换 `chart_entry_bar`。保护数量、实际仓位、实际成交、部分成交、回执不明、平台拒绝和连接异常属于执行层事实与安全分支，统一按[执行手册](../execution/execution_manual.md#五行动前锁定计划)写入执行记录。本页只要求在锁定策略前确认该计划能够被当前环境安全实现，不另行定义或放宽这些操作规则。

共同时间门槛在候选合格、请求入场和图表触发前都必须成立。到达 `latest_entry_time` 时，所有未成交入场余量的策略资格失效；到达 `forced_exit_start` 时，计划触发“不再增加风险并开始退出”的意图。撤单、退出、订单与仓位核对的真实动作和安全顺序只由执行手册规定。完整时间字段及跨 session 边界见[策略共同规则](common_rules.md#时间与持有门槛)。

计划锁定后，不得因为害怕错过而追价，不得扩大止损，不得临时增加仓位，也不得把目标换到更远位置来改善纸面回报。

未成交前若价格触及或越过计划保护止损价，原结构已经不再支持该入场：未成交入场资格立即失效并结束计划，不等待收盘，也不把止损向外移动；执行层据此请求撤单并核对真实状态。保护止损成交则直接记为 premise 失效；页面另写的提前退出只是在止损以前可能出现的附加路径。

## K 线计数的共同约定

策略页提到“之后第几根完成 K 线”时，使用以下约定：

- 形成事件的 K 线是参照 K 线，不计入之后的根数。
- 参照 K 线之后第一根完整收盘的 K 线算第一根。
- “三根以内”包括第一、第二和第三根；第三根收盘仍未满足条件，机会过期。
- “等待两根仍未成交”从入场单第一次有效挂出后的下一根完整 K 线开始计数；第二根收盘仍有未成交余量时撤销余量。部分成交不重置计数，已经成交的仓位继续按原计划管理。
- K 线跨越预先写定的时段截止时间时，以策略页规定的判断时点为准；没有写清时不执行。

数据缺失、周期切换或顺序无法唯一还原时，不凭结果重新计数。

## 交易理由的三种变化

- **仍然成立**：新价格仍与原计划相容，继续按计划管理。
- **已经削弱**：支持证据减少，但还没有达到策略页写明的失效事实。只执行计划中预先写明的动作。
- **已经失效**：否定原交易理由的价格事实已经出现。未成交入场资格终止；已有持仓触发预写退出意图。撤单、退出、保护与仓位归零均由执行层按平台事实完成。

交易理由失效后，原记录不能改回“仍然成立”。后来出现的新机会另行记录。

## 同一机会不得重复承担风险

强突破、突破跟进和突破回测可能依次出现；首次恢复与二次恢复也可能属于同一次回调。只要旧计划仍可能成交或仍有持仓，就不能再用相邻策略建立第二份风险。

准备切换进入时点时，先明确结束旧计划，再从当前价格重新计算新策略的入场、止损、目标、数量和剩余空间。

## 最后检查

- [ ] 市场家族和位置能够用完整句子说明。
- [ ] Market cycle 不清时只采用了逐层 fallback，没有把保守管理假设当成区间或其他策略许可。
- [ ] 开盘或尾盘覆盖窗口内，已经先检查时段页，并且没有用普通页规避时段限制。
- [ ] 只选择了一项策略。
- [ ] Always In 的周期、来源事件、当前方向与反证已经记录，且没有把方向当成实际持仓或入场许可。
- [ ] 形成顺序只使用当时可见的信息。
- [ ] Signal bar、chart entry trigger / bar 与账户 actual fill bar 已分开；历史回放或模拟另记 simulated fill bar，尚未发生的图表触发或成交保持为空。
- [ ] 入场、撤单、止损、目标区域、页面基线目标、实际退出价格、管理和失效条件已经写清。
- [ ] Remaining holding time、latest entry 与 forced exit 三项时间门槛同时成立。
- [ ] 已知日程事件、基础设施可用性与适用尾部风险已经进入计划；无法维持保护或安全退出时没有承担新风险。
- [ ] 结构止损和现实目标已经先确定；成本、保守滑点、净风险、净回报、净 R、最坏风险和前方障碍已经按共同口径计算，Trader's Equation 仍然有利。

# 人工策略手册

> **定位：供交易者根据当前市场选择一项完整策略**
>
> 本目录只回答“看到什么、什么时候行动、什么时候放弃以及怎样管理”。基础概念回到[核心框架](../core/README.md)，订单和成交安全回到[执行手册](../execution/execution_manual.md)。

策略页底部的“概念关联”只标出该页直接引用的定义，不构成缩减版理解清单。使用任何策略前，都应完整理解 `core/` 的全部组成及其关系。

## 选择入口与唯一规则

本页只提供策略目录，不定义第二套选择顺序。市场状态的逐层 fallback、时段覆盖、单候选约束、计划锁定、结束与重新判断，全部以[如何选择并锁定一项策略](how_to_choose.md)为全局权威状态机；路由进入家族后，家族内分类与重叠优先级由对应 README 负责，具体资格与参数由所选策略页负责。

简要入口是：先判断当前周期的基础状态与控制证据，再检查时段覆盖，最后在下列家族中保留一项候选。状态不清时采用 Core 的保守管理假设，但这不会自动授权任何区间、突破或反转策略；具体候选仍须独立满足所选页面的全部条件。

## 时段页是覆盖层

[开盘与尾盘覆盖层](session/README.md)在普通市场家族之上增加候选起点、最晚入场、目标和强制退出约束。同一底层机会发生覆盖、冲突、过期或切换时，只按[选择状态机的时段覆盖规则](how_to_choose.md#时段策略怎样覆盖普通策略)处理；不能从本目录摘要推导例外。

## 策略共同基线

[策略共同规则](common_rules.md)是价格、四类风险、时间门槛、页面基线目标、实际退出价格、净 R、等待失效和管理口径的唯一权威位置。全部正式策略页继承这套基线；具体策略页只有明确写出时才可采用更严格要求或局部覆盖。

仓库选择结构保护止损和单一实际全量退出价格作为固定订单基线，标准计划仍保留预写的 premise-change 主动退出；`mechanical_only` 只能作为独立研究变体。价格行为概念回到 `core/`，订单回执、成交、仓位与保护等安全事实始终回到[执行手册](../execution/execution_manual.md)，不受策略层覆盖。

## 同一次机会只选择一项

相邻策略可能描述同一段价格运动的不同进入时点，但不能据此建立多份风险。候选、锁定、结束和重新选择统一按[全局选择状态机](how_to_choose.md#唯一选择与覆盖状态机)处理；原计划失败也不自动构成反手理由，见[原计划失败后如何重新判断](switching_after_failure.md)。

## 每个策略页都应回答什么

每一页只写一套完整计划。选择和计划语义字段由[如何选择策略](how_to_choose.md)统一规定；价格、目标、风险、时间与管理接口由[策略共同规则](common_rules.md)统一规定。具体页只实例化该策略的环境、形成顺序、入场、有效期、保护、目标、失效与特有风险，不能另造默认口径。

这些页面是供人执行的仓库整理。执行前仍须设置具体市场、周期、时段、成本、计划 account risk 边界，以及限制 Personal risk 的纪律护栏。

页面中的固定根数、观察窗口、入场时点和管理方法，是本仓库为了让同一规则能够一致执行而作出的人工基线选择，不应冒充 Brooks 适用于所有市场和周期的通用硬规则。

# 人工策略手册

> **定位：供交易者根据当前市场选择一项完整策略**
>
> 本目录只回答“看到什么、什么时候行动、什么时候放弃以及怎样管理”。基础概念回到[学习路径](../learning/README.md)，订单和成交安全回到[执行手册](../execution/execution_manual.md)。

## 先选择市场家族

选择策略前，先用当时可见的信息回答：

1. 市场更接近趋势、交易区间、双向等待、趋势反转过程，还是暂时看不清？
2. 如果是趋势，当前机会来自强突破、趋势内回调，还是成熟趋势的控制权转移？
3. 如果是区间，价格是否真正位于边缘？区间中部通常不选择区间反转策略。
4. 当前是否处于开盘或尾盘等会实质改变目标和剩余时间的时段？
5. 前方空间、合理止损和成本是否允许形成一份完整计划？

看不清时不强行分类。完整选择过程见[如何选择策略](how_to_choose.md)。

## 同一次机会只选择一项

相邻策略可能描述同一段价格运动的不同进入时点。例如一轮突破可能先出现强突破收盘，随后出现跟进，再出现第一次回测。它们是三种不同计划，不是三次自动加仓理由。

在机会开始前或第一项条件出现时，写清准备等待哪一种进入方式。已有活动计划时，不得因为后续又出现另一个策略名称而重复承担风险。若原计划结束，必须从当前价格重新检查另一项策略的全部条件。

原计划失败也不自动构成反手理由，见[原计划失败后如何重新判断](switching_after_failure.md)。

## 每个策略页都应回答什么

每一页只写一套完整计划，并明确：

- 交易的市场行为；
- 适用和不适用环境；
- 机会形成顺序；
- 条件完整与放弃边界；
- 入场、撤单、保护止损、目标和持仓管理；
- 成交后的正常波动与前提失效；
- 策略特有的订单风险；
- 正例、反例、边界案例和复盘问题。

这些页面是供人执行的仓库整理。执行前仍须设置具体市场、周期、时段、成本、账户和个人风险边界。

页面中的固定根数、观察窗口、入场时点和管理方法，是本仓库为了让同一规则能够一致执行而作出的人工基线选择，不应冒充 Brooks 适用于所有市场和周期的通用硬规则。

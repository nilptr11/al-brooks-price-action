# 人工策略手册

> **定位：供交易者根据当前市场选择一项完整策略**
>
> 本目录只回答“看到什么、什么时候行动、什么时候放弃以及怎样管理”。基础概念回到[学习路径](../learning/README.md)，订单和成交安全回到[执行手册](../execution/execution_manual.md)。

## 先选择市场家族

选择策略前，先用当时可见的信息回答：

1. 基础状态更接近趋势还是交易区间，还是暂时看不清？
2. 如果是趋势，当前更像强突破还是通道；机会来自趋势内回调，还是成熟趋势的控制权转移？
3. 如果是区间，当前是边缘交易、双向等待还是已经出现突破过程？区间中部通常不选择区间反转策略。
4. 当前是否处于开盘或尾盘覆盖窗口，因而必须先检查时段页特有的截止时间、目标和退出要求？
5. 前方空间、合理止损和成本是否允许形成一份完整计划？

看不清时不强行分类。完整选择过程见[如何选择策略](how_to_choose.md)。

## 时段页是覆盖层，不是规避入口

趋势与交易区间是基础状态；突破或通道细化趋势阶段，双向等待和趋势反转描述 setup 状态或控制权转移过程。开盘与尾盘页在这些判断之上增加时间、目标和剩余风险约束。进入预先写定的开盘或尾盘窗口时，先检查[开盘与尾盘覆盖层](opening/README.md)，再决定当前机会是否还能进入普通策略家族。

- 同一个底层交易机会同时落入时段页和普通页时，使用时段页。“同一个”指普通页拟议入场与时段候选依赖同一参照事件、预先边界和方向；即使等待的信号或入场时点不同，仍是同一机会。时段页的截止时间、目标和退出要求优先，不得选择普通趋势、突破、区间或反转页绕过这些约束。
- 时段覆盖从具体页面规定的候选起点开始，只持续到该候选或计划被拒绝、失效、过期、未成交或结束；仅仅处于开盘或尾盘窗口，不会锁定无关的普通机会。一次只保留一项活动候选或计划。同一底层机会若同时满足两张时段页，而主要交易前提无法唯一确定，就继续等待或放弃，不用更有利的名称替自己选择。
- 时段候选或计划被拒绝、失效、过期、未成交或结束后，从当前价格重新判断。原信号、原触发和原订单不能改名或复用；新候选必须出现旧计划结束后才完成的当前结构或触发，并独立满足所选页面全部条件。已经完成的突破、接受、趋势或区间结构可以作为页面明确要求的历史背景。
- 时段页条件不完整或过期不会自动生成普通策略。若拟议普通入场仍依赖原时段候选的参照事件、边界和方向，却没有在旧候选结束后形成新结构或新触发，就直接放弃；只有新的当前机会独立合格时，才按普通页处理。

具体判断边界见[如何选择策略：时段策略怎样覆盖普通策略](how_to_choose.md#时段策略怎样覆盖普通策略)。

## 所有策略共同使用的价格与净 R 口径

以下规则适用于本目录全部正式策略页；具体页面写得更严格时，以页面为准。

| 口径 | 回答的问题 | 使用方式 |
| --- | --- | --- |
| 目标区域 | 当前 setup 为什么可能测试这里？ | 来自当时可见的结构、支撑阻力或 magnet，例如旧高低、区间内部目标或量度投射附近；它不承诺价格精确触及某一跳 |
| 页面基线目标 | 这一页统一使用哪个结构参照或公式？ | 为同类交易保持一致；它是仓库选择，不穷尽 Brooks 对该形态的其他合理管理方式 |
| 实际退出价格 | 订单提交在哪个可交易价格，计划回报按哪里计算？ | 必须在目标区域内或沿交易方向的近侧，并按最小跳动、流动性和预先写定的成交余量确定 |

- 本手册所说的“固定”或“冻结”，不是所有交易共用一个价格、点数或 R。它表示当前机会已经根据当时可见的 setup、结构端点和 magnet 得到具体价位；计划形成后不再为迎合后来行情而重画。新的机会必须重新分析并得到自己的入场、止损和目标。
- 具体策略页若写明“唯一目标／固定目标是某价格，并使用全量限价退出”，按该页公式和取整规则得到的可交易价格同时是页面基线目标与实际退出价格，不再临场另选一点。只有页面仅给出目标区域而没有给出唯一价格时，才在计划形成时选择近侧退出价：做多取区域下沿或其入场侧，做空取区域上沿或其入场侧，并记录具体价格。
- 目标区域、页面基线目标和实际退出价格必须来自同一个 setup 与同一组事先可见的结构端点。候选形成以后，不得为提高纸面净 R 而扩大目标区域、改量另一段腿、改选更远 magnet 或使用更有利的区域边缘；也不得把途中一个原本不是本页目标的近端障碍临时改名为目标来挽救计划。页面规定的 premise 失效退出属于持仓管理，不是修改目标。
- 计划形成时必须把目标区域及来源、页面基线目标、实际退出价格、参照边界、计划入场价或保守预计成交价、保护止损价、数量、每跳价值、往返费用和保守滑点写成具体数值。只写“附近”“结构外”或“空间足够”不算计划完整。
- **计划净风险金额**先把保守预计成交价到保护止损的价格距离按最小跳动、每跳价值和数量折算成金额，再加上止损侧预计滑点金额，以及入场和止损退出的全部费用。**计划净回报金额**先把保守预计成交价到实际退出价格的价格距离按同一方法折算成金额，再减去目标侧预计滑点金额，以及入场和目标退出的全部费用；不得使用更远的页面基线目标虚增回报。保守预计成交价已经包含入场侧滑点假设，不重复计算；费用先按对应订单和数量汇总为金额，汇总后不再乘以每跳价值或数量。计划净 R 等于计划净回报金额除以计划净风险金额。存在多层或不同数量时，分别计算每层金额再相加，不能只用未经费用调整的平均价代替。
- Brooks 语境中的“约 2R”通常描述图表上潜在 reward 相对 initial price-action risk 的关系；本手册的计划净 R 还包含费用和保守滑点。两者不能直接等同，更不能把语境性参考改写成固定的成本后净 R 门槛。
- 计划净 R 是结构分析完成后的记录值，不是用来反推价位的预设参数，也不是统一入场门槛。先按当前 context 与 setup 确定入场、结构保护止损、现实目标和实际退出价格，再计算净风险、净回报与净 R；不得为了得到想要的 R 而缩窄止损、扩大目标或更换结构锚点。
- 本手册把“净空间足够”定义为：实际退出位于盈利方向，扣除费用和保守滑点后仍有正净回报，目标在当前路径上合理可达，并且把该 setup 的概率、止损、回报、途中障碍和管理方式放入同一个 Trader's Equation 后仍然有利。净 R 只是这个方程中的风险回报描述，不是胜率估计；较高净 R 不能补足背景、位置或形成顺序，较低净 R 也不能脱离概率单独否定一笔交易。
- 从保守预计成交价到目标区域之间若有事先可见的强支撑、阻力或 magnet，必须记录其具体价格和来源，并按实际退出价格以及该障碍影响后的到达概率重新检查 Trader's Equation。障碍使目标区域不可合理到达或交易者方程不再有利时，入场前放弃；只有策略页本来就把该结构定义为目标时，它才属于目标区域，不能在候选出现后临时改名。
- 未成交订单仍在等待时，价格一旦触及或越过计划保护止损价，立即撤销入场并结束旧计划，不等待收盘确认，也不把止损移到新极值外。唯一例外是尚未选择方向的 ii 原子二选一计划：母线另一侧同时是相反方向入场触发，按该页的联动规则处理；任一方向成交后，本条立即适用。若入场和失效价格可能在同一根 K 线或平台竞态中先后触发，先核对真实成交和仓位，按执行异常恢复保护。
- 策略页规定的未成交有效期同样约束部分成交后的剩余入场数量。部分成交不重置计数，也不延长订单寿命；到期时撤销全部未成交余量，已经成交的仓位继续按原页管理。目标成交、保护止损成交或主动退出流程开始时，所有剩余入场数量立即失去继续增加风险的资格，按平台事实撤销并核对。
- 页面列明的结构保护止损成交本身表示所选策略依赖的结构已经失效，即 premise 失效。页面规定的收盘型提前退出只是可能更早发生的额外失效路径；仓位确认归零以前，结构保护止损始终有效。
- 结构保护止损与更远的 catastrophe stop 不是同一概念。本手册正式策略页列出的都是结构保护止损；页面写“不使用视觉提前退出”时，表示只按该结构止损和固定目标管理，不表示另有一个更远的灾难性止损可以替代 premise 管理。若交易者另设更远的 catastrophe stop 作为平台后备保护，必须同时预先写清 premise 失效时的提前退出，不能被动等待该宽止损成交。概念区别见[Stop Entry 和 Protective Stop](../learning/03_order_flow/01_stop_entry_vs_protective_stop.md)。

除非具体页面明确另行规定，本仓库使用一个页面基线目标和一个实际全量退出价格，不分批止盈、不留 runner。Brooks 的目标与 magnet 思想允许按背景选择目标区域，并可能使用部分兑现或 runner；这里的单目标、近侧定价、净 R、取整和等待失效规则，是仓库为一致执行设定的人工基线，不应冒充 Brooks 对所有市场和交易者规定的统一参数。

## 同一次机会只选择一项

相邻策略可能描述同一段价格运动的不同进入时点。例如一轮突破可能先出现强突破收盘，随后出现跟进，再出现第一次回测。它们是三种不同计划，不是三次自动加仓理由。

在机会开始前或第一项条件出现时，写清准备等待哪一种进入方式。已有活动计划时，不得因为后续又出现另一个策略名称而重复承担风险。若原计划结束，必须从当前价格重新检查另一项策略的全部条件。

原计划失败也不自动构成反手理由，见[原计划失败后如何重新判断](switching_after_failure.md)。

## 每个策略页都应回答什么

每一页只写一套完整计划，并明确：

- 交易的市场行为；
- 适用和不适用环境；
- 机会形成顺序；
- 条件完整与放弃边界；
- 入场、撤单、保护止损、目标和持仓管理；
- 成交后的正常波动与前提失效；
- 策略特有的订单风险；
- 正例、反例、边界案例和复盘问题。

这些页面是供人执行的仓库整理。执行前仍须设置具体市场、周期、时段、成本、账户和个人风险边界。

页面中的固定根数、观察窗口、入场时点和管理方法，是本仓库为了让同一规则能够一致执行而作出的人工基线选择，不应冒充 Brooks 适用于所有市场和周期的通用硬规则。

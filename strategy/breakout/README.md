# 突破延续策略

本目录交易的是重要边界外的新价格正在获得接受。高点或低点越过重要边界已经构成最低突破事件，但尚未达到本目录的可交易门槛；只有收盘、跟进、回测守住或拒绝过程再次失败，才形成这里的具体计划。

**来源层**：breakout、follow-through、breakout test、failure / failed failure 与 measured move 来自 Brooks；本目录的 K 线比例、严格下一根比较、三根或十根窗口、两根订单寿命、结构高度单一投射与全量退出，是仓库执行基线。策略页只定义可执行子型，不覆盖 Core 与来源台账中的通用概念。

本页只在[全局选择状态机](../how_to_choose.md)路由到突破家族后，负责本家族内部分类与重叠优先级；具体资格、触发、参数和特有失效以所选策略页为准。本页不改写时段覆盖、单候选、结束或重入规则。

本目录的四项策略都必须绑定明确的交易周期和观察窗口。单次越界不自动建立趋势；接受证据足以形成方向控制时，当前阶段才可能升级为 breakout phase。该阶段出现第一段可识别回调后，阶段标签转入 channel，但突破延续交易仍可通过跟进、边界测试或 failed failure 继续成立。后来若出现新的强突破阶段，必须从新结构重新确定边界、entry、stop、target 和 Trader's Equation，不能沿用旧阶段的计划输入。

## 开始前必须有旧边界

突破 K 线形成以前，先写清被突破的边界及其来源。边界可以来自已经形成的交易区间、旗形或清楚摆动，但不能在看到结果后移动到最有利的位置。

若当前基础状态仍是成熟交易区间，用来宣称“离开区间”的边界必须是事前固定的区间外沿。完全发生在区间内部的强第一腿、第二腿，或只越过内部小摆动的运动，即使外观很强，也不能借用“新价格已在区间外获得接受”的突破延续逻辑；只有真正越过外沿并在外部获得接受后，才重新路由到本家族。否则继续按双向轮动或[区间第二腿陷阱](../../core/01_market_cycle/02_trading_ranges.md#强区间腿与第二腿陷阱)判断，第二腿本身也不自动构成反向信号。

以下情况不使用本目录：

- 边界是当前突破 K 线出现后才画出的；
- 在成熟区间内部只越过局部小边界，尚未真正离开并守住区间外沿；
- 价格位于没有明确边界的普通趋势中部；
- 目标以前存在更近且明显的强障碍；
- 合理止损过远，或扣除成本后没有可执行空间；
- 价格已经重新进入旧区域并持续获得反向跟进，此时应重新判断，而不是坚持突破延续。

## 四种进入时机

| 愿意等待到什么程度 | 使用的策略 |
| --- | --- |
| 强突破 K 线完成后，在下一根开盘直接跟随 | [强突破收盘后立即跟随](strong_breakout_close.md) |
| 只交易紧接的一根严格子型：仍在边界外，且收盘沿突破方向超过突破 K 线收盘 | [紧接突破的一根延伸收盘确认](breakout_follow_through.md) |
| 等突破获得接受，再交易第一次回到边界的测试 | [突破后的第一次回测](breakout_first_pullback.md) |
| 边界外收盘短期收回旧区后，再次接受原突破方向 | [收盘突破被拒绝后再次接受原方向](failed_failure_continuation.md) |

这四张页面不是同一机会上的四次加仓理由。同一次突破只能选择一种进入方式。较早进入确认少、价格较好；等待越久，确认可能更多，但入场更差、止损可能更远、剩余目标也更近。

### 与 Buy / Sell The Close 的边界

Core 中的 Buy The Close / Sell The Close 指在强收盘附近承担风险，可以是首根强 breakout bar 的预测／早期版本，也可以是连续强收盘或 follow-through 已出现后的确认版本。它描述风险承担时点，不是独立 Setup。

本目录目前没有把字面意义上的 BTC / STC 单独实例化为策略页：

- [强突破收盘后立即跟随](strong_breakout_close.md)在突破 K 线完成后作决定，但实际 entry 固定为下一根开盘，因此是仓库的 next-open 版本，不能把突破收盘价当成真实或预计成交价；
- [紧接突破的一根延伸收盘确认](breakout_follow_through.md)等待 follow-through 完成后，再用其极值外的 stop entry 触发，因此也不是确认版 BTC / STC；
- 第一次回测和 failed failure 页承担的是更晚的确认风险，同样不能借用较早收盘附近的价格。

若计划实际在强收盘附近承担风险，必须另有完整策略实例写清该时点自己的 entry、完整结构 stop、剩余 target、成本和 Trader's Equation；不能把上述页面改名后复用其参数。

通用概念与本目录严格门槛必须分开：

| 通用概念 | 本目录怎样收窄 |
| --- | --- |
| Follow-through 可以由一根或多根延伸构成，也不统一要求紧接出现 | 跟进页只交易紧接第一根、边界外、收盘进一步延伸且在自身方向半部的子型 |
| Breakout test 可以很快，也可以延后很多根 | 第一次回测页只接受自己的十根窗口和下一根恢复信号 |
| Glossary failure 取决于 stop 与 scalp／原目标的结果顺序 | 收回旧区域只作为拒绝或可能失败观感的代理，不单独确认 glossary failure |
| Failed failure 是先前失败再失败并恢复原方向的机制 | 恢复页只交易“外侧收盘—三根内收回—三根内再次外侧收盘”的仓库子型 |

成交后三张普通突破页使用连续两根收回旧区域的额外 premise 失效；恢复页因市场已经展示过一次拒绝，使用任意一根完成 K 线再次收回的更严格阈值。这个差异来自页面 premise，不得互相复制。

## 突破家族特有约束

价格、净 R、等待失效、保护与固定管理的通用语义只由[策略共同规则](../common_rules.md)定义。下列内容只补充突破家族的边界与目标算法：

- 向上和向下突破完全镜像；
- 强突破收盘立即跟随页使用合格强突破 K 线的完整高度；其余三页使用突破前已经画定的区间、旗形或其他清楚结构的完整高度，并从被突破边界向外投射一次；
- 上述投射是本目录从 Brooks measured-move 候选中固定的一种基线目标，不表示每次突破只有这一种合理投射，也不保证价格到达；各页已经把公式所得可交易价格同时写成实际全量限价退出价。途中强障碍使计划不合格时放弃，不在成交后改用较近障碍挽救计划，也不改用另一段距离延长目标；
- 使用结构高度的页面必须在突破以前写清结构的两侧边界；只有单一摆动位、无法确定被突破结构高度时，不使用那三页；
- 目标被计划入场越过，或目标前有更近的强障碍时，放弃；
- 收回边界不必自动等同于失败，但页面规定的重新接受旧区域条件一旦出现，就按页面处理；
- 原突破交易失败不意味着自动反手。反方向必须独立满足另一张正式策略页；
- BTC / STC 的早期版本不能借用后续 follow-through 的概率，确认版本也不能继承早期版本更好的 entry；
- 订单、成交、所选环境确认的仓位和保护操作统一遵守[人工交易执行手册](../../execution/execution_manual.md)。

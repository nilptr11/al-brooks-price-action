# 交易区间策略

> **定位：供交易者在成熟宽区间的边缘选择一种反转进入方式**

本家族继承[策略共同规则](../common_rules.md)。限价试仓页的两层预设入场是明确的局部例外；它只覆盖该页写出的数量与入场字段，不建立第二套净 R、保护或执行安全规则。

**来源层**：Brooks 支持成熟宽区间的 BLSHS、边缘反转、失败突破、结构止损和有经验者的预设 scale-in；本目录的“两侧各两次”“两次穿越中部”、精确算术中点、根数窗口、两根订单寿命、两层等量与单一全量退出，是仓库执行基线，不是 Brooks 对所有 trading range 的统一参数。

本页只在[全局选择状态机](../how_to_choose.md)路由到区间家族后，负责本家族内部分类与重叠优先级；具体资格、触发、参数和特有失效以所选策略页为准。本页不改写时段覆盖、单候选、结束或重入规则。

## 什么时候进入这个家族

交易区间策略依赖旧公平区域继续主导。最低背景是：

- 价格已经多次从上下两侧反转，而不是刚出现第一次横向停顿；
- 双向 K 线和重叠增多，至少多次穿越区间中部；
- 上沿、下沿和中部能够在机会出现前画出；
- 当前位于外侧边缘，而不是区间中部；
- 边缘到按本目录公式得到的中部目标之间，扣除成本后仍有正净回报；目标合理可达，且结合区间背景、概率、结构止损和途中障碍后 Trader's Equation 有利。

区间内可以出现很强的单腿。只有新价格在边界外获得收盘、跟进和持续接受，才离开本家族转向突破策略。

## 剩余时段门槛

区间回归依赖价格有时间重新靠近旧公平区域。每次机会成为候选以前，必须把计划持有周期、最晚允许入场时点、最迟开始强制退出的时点，以及计划 account risk 的最坏持有情景金额写成明确计划字段；它们不能等成交后再由盈亏决定。

最坏持有情景至少比较两条路径：一是结构保护止损先成交；二是价格始终没有回到目标、也没有先触及保护止损，而是在不利区域停留到强制退出时点，再按预设压力价格、退出滑点和费用全量结束。两条路径取金额较大者，作为计划 account risk 的保守情景金额并决定仓位资格。它不是四类风险以外的第五种风险；压力价格也只用于保守预算时间退出，不是新增止损或事后可修改的目标。

对必须在本 session 内结束的计划，还须满足以下边界：

- 最晚入场与强制退出之间仍有足够时间让本页目标合理可达；只剩图上价格空间、却缺少实现时间，不能成为候选。
- 条件在最晚入场时点或之后才完整时直接放弃；未成交余量的策略资格到该时点终止，不能把旧边缘测试留给下一段时窗。实际撤单与状态核对按执行手册完成。
- 临近收盘不得依赖宽结构止损、逆向限价或继续加仓来等待均值回归；到强制退出时点，即使结构止损尚未触发，也必须触发预写退出意图，实际退出与保护顺序按执行手册完成。

允许跨 session 的计划也必须预先写明具体持有边界和强制退出时点，并把隔夜跳空、结算、流动性与退出滑点纳入计划 account risk 的最坏持有情景；不能把原日内计划临时改成隔夜计划。

## 固定边界与区间中部

三张区间策略共用同一个中部算法：机会成为候选以前，把成熟区间上沿记为一个固定可交易价格、下沿记为另一个固定可交易价格；原始中部等于“上沿加下沿再除以二”。原始中部不在最小跳动上时，目标向计划入场一侧取整，使目标更近而不是更远：下沿做多向下取到最近可交易价格，上沿做空向上取到最近可交易价格。

候选形成以后，上沿、下沿和中部全部冻结。本轮不得把“中部”解释成中央价值区、视觉中心、区间中间三分之一中的任意价格，也不得因后来扩张重新计算。固定中部不能提供正净回报、其前方有使目标不可合理到达的强障碍，或完整 Trader's Equation 不再有利时，放弃而不是更换目标。

精确算术中点是本仓库为了让区间边缘向内 scalp 得到唯一可挂单价格而选择的保守基线，不是 Brooks 对所有 trading-range fade 规定的统一目标。Brooks 的上层原则是 BLSHS、避开中部入场，并根据区间宽度、概率、止损和途中磁铁选择现实利润；本目录只固定其中一种管理方案，不把中点到达写成保证。

## 三项策略怎样选择

| 当前路径 | 只选择这一项 |
| --- | --- |
| 尚未出现边界外收盘，边缘出现明确反向信号 | [成熟区间边缘确认后反转](range_edge_confirmation.md) |
| 尚未出现边界外收盘，事先决定以较小首仓和两层限价处理宽区间 | [成熟区间边缘限价试仓与预设加仓](range_edge_limit_scale_in.md) |
| 已经收在边界外，随后重新进入旧区间 | [突破失败后回到区间](failed_breakout_return.md) |

确认入场和限价加仓的成交价格、概率、止损和仓位逻辑不同，同一次边缘测试只能选择一种。边界外收盘或持续接受出现后，也不能继续把它当作普通边缘触碰。

三页成交后的主动失效阈值也不同，原因是入场 premise 不同：

| 页面 | 主动失效阈值 | 结构理由 |
| --- | --- | --- |
| 边缘确认 | 一根完成 K 线收在边界外 | 入场前提原本就是测试期间没有外侧收盘 |
| 失败突破回区间 | 连续两根重新收在外侧，第二根沿突破方向延伸 | 允许一次边界重测，等待外部价格重新获得接受 |
| 限价试仓与预设加仓 | 强外侧收盘后再有一根沿突破方向跟进 | 计划预先容忍普通越界，只在强突破与跟进时提前结束 |

这些阈值是三张仓库策略各自的执行分支，不能互相复制；一根外侧收盘可以结束“边缘确认”页，但不自动证明趋势突破已经获得接受。

## 扩张三角形怎样处理

扩张三角形是高点更高、低点更低的波动扩大型区间，不是压缩结构。只有外侧摆动能够事先画定、区间仍保持双向交易并且目标空间合理时，才使用本家族：

- 外侧边缘出现反向确认，使用边缘确认策略；
- 事先具备宽止损和两层计划，使用限价试仓策略；
- 越界收盘后重新进入，使用失败突破策略；
- 形态相对整张图尺度足够、没有受困于紧密区间，且第四或第五腿已经形成足够末端反向压力时，可以离开普通区间 fade 路由，转入[MTR 家族](../reversal/README.md)的候选检查；这只建立候选，仍须重新满足成熟原趋势、通道或趋势线破坏、旧趋势恢复测试失败和所选 MTR 页的完整前序，不直接授权反转入场；
- 突破获得接受，离开本家族并重新判断突破策略。

## 什么时候离开这个家族

- 区间还不成熟，边界只能事后画出；
- 当前位于区间中部；
- 区间太紧，扣除成本后没有现实利润；
- 边界外出现强突破和跟进；
- 合理结构止损超出本次风险边界；
- 大周期重要位置使原区间目标失去意义。

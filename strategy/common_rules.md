# 策略共同规则

> **状态：Strategy / Common Baseline**
>
> **定位：策略层对价格、风险、时间、等待失效与管理分支的共同权威页**

本页是 `strategy/` 全部正式策略的共同基线。具体策略页是这套基线的可执行实例：没有明确写出更严格要求或局部覆盖时，必须继承本页；若具体页确实覆盖某项规则，必须明确指出被覆盖的字段和本页基线，不能靠含混措辞产生第二套默认口径。

本页不选择策略，也不处理时段页与普通页的优先级；这些状态转换只由[如何选择并锁定一项策略](how_to_choose.md)规定。

[核心目标专题](../core/02_context/01_support_resistance_targets.md)只定义 target、magnet、路径障碍和 measured move 等价格行为概念。订单是否送达、生效、部分成交、撤改单、仓位和保护是否真实有效等执行事实，仍以[执行手册](../execution/execution_manual.md)为准，任何策略页都不能覆盖或弱化这些安全边界。

本页采用结构保护止损、一个页面基线目标和一个实际全量退出价格，作为仓库为一致执行选择的固定订单基线；它不等于“只等止损或目标”的完整管理。标准策略同时必须预写原 premise 被强反向证据否定时的主动退出。只隔离 stop / target 结果的 `mechanical_only` 是另行声明的研究变体，不是 Brooks 对所有市场和交易者规定的管理方式。执行异常也不能被动等待预定止损或目标；回执不明、保护不足、意外成交、平台拒绝或连接异常等情况必须按执行手册恢复事实核对和仓位保护。

以下规则适用于本目录全部正式策略页；具体页面若显式写出更严格要求或局部例外，以该页面的具体实例为准。没有点明覆盖字段的近似说法，不构成例外。

## 风险字段与事前边界

策略计划继承[四种不能混用的风险](../core/00_method/01_probability_risk_reward.md#四种不能混用的风险)，并按下表保留各自职责：

| 风险字段 | 策略层用途 | 时间边界 |
| --- | --- | --- |
| Initial / price risk | 计划 entry 到合理结构保护止损的价格距离；用于事前 Trader's Equation 和仓位计算 | 承担风险前确定 |
| Actual Risk / MAE | 价格在一笔已结束交易中实际走过的最大不利距离；用于连续样本复盘 | 只能事后得到 |
| 计划 account risk | Initial / price risk 乘计划数量与每点价值，再计全部计划加仓、费用和保守滑点后的金额暴露 | 事前估算 |
| Personal risk | 因希望、恐惧、仓位过大、心理止损或破坏规则而产生的额外损失路径 | 承担风险前设置护栏，执行中持续限制 |

- 事前计划、Trader's Equation 和仓位必须使用 Initial / price risk 与计划 account risk。Personal risk 不是可与前三者并列代入方程的价格距离或单一金额；计划要用账户亏损上限、合适仓位、硬保护、不得扩大止损或临时加仓等纪律护栏限制它。不得用赢家事后较小的 Actual Risk / MAE 缩窄原风险、提高原净 R 或证明原入场更优。
- Actual Risk / MAE 只作为完整连续样本的复盘字段。它可以支持以后研究另一套止损或管理方案，但不能覆盖原计划的 entry、保护止损和失败路径。
- 本页的“计划净风险金额”属于计划 account risk 的成本后表达。真实成交、实际数量、当前仓位和实际账户暴露由[执行手册](../execution/execution_manual.md#八成交以后先更新仓位)根据平台事实更新；这些执行结果不能反向改写原计划输入。
- 允许 scale-in 的页面必须在第一层以前把全部计划层纳入最坏 account risk；各层的风险数学回到[加仓与减仓](../core/06_trade_plan_and_management/02_scaling_in_out.md)，不能用更好的预期均价忽略未成交层的潜在暴露。

## 时间与持有门槛

以下命名字段是仓库为了让时间风险可以执行和复盘而采用的计划接口，不是 Brooks 官方给出的固定字段或跨市场阈值；具体数值必须随品种、周期、session 和计划持有方式确定。

每份策略计划都必须同时声明三个时间字段：

| 时间字段 | 必须回答的问题 |
| --- | --- |
| `minimum_remaining_holding_time` | 从可能成交到开始强制退出之间，至少还需多少时间或完整 K 线，才能容纳该 setup 的正常波动、目标路径和预写管理？ |
| `latest_entry_time` | 最迟何时仍可承担新风险？到达该时点后，未成交入场余量必须失去有效资格 |
| `forced_exit_start` | 最迟何时开始撤销剩余入场、结束目标等待并按计划退出已有仓位？ |

- `latest_entry_time` 必须严格早于 `forced_exit_start`，并预留策略所需的最低持有时间以及合理执行缓冲。计划时实际可用的剩余持有时间，从保守预计成交时点计算到 `forced_exit_start`；它小于 `minimum_remaining_holding_time` 时，候选不合格。
- 这三项在候选合格、订单提交和图表触发前都要重新成立。较晚才完成的信号、延迟确认或滑点不能借用较早候选时的剩余时间；到达最晚入场时点仍未成交，不得把订单留到强制退出窗口。
- 每份计划还必须预写“截止后晚到成交”分支。若所选环境能证明成交发生在 `latest_entry_time` 以前，且成交时三项门槛仍成立，按原计划管理；若成交发生在截止以后，或无法证明发生时间早于截止，该成交不重新获得策略资格，必须作为执行异常先保护已确认仓位、停止新增风险并立即触发退出。撤单请求、回执与退出安全顺序只由执行手册规定。
- 具体时段页给出更早截止或更长退出缓冲时，以更严格字段为准。普通策略也不能因为没有时段页覆盖就省略时间门槛。
- 允许跨 session 的计划必须在承担风险前另行写定持仓边界、隔夜风险和计划级 `forced_exit_start`；“可跨 session”不等于无限期持有，也不能在成交后临时取消原时间边界。
- 本节只规定策略必须提供的时间接口。实际时钟、订单状态、撤单确认、退出成交和仓位归零均按[执行手册](../execution/execution_manual.md#十一交易理由发生变化)核对，不能用计划时间替代平台事实。

## 外部事件、基础设施与尾部风险

价格反应仍是判断方向、接受与失败的交易证据；新闻标题、日程事件或基础设施状态不能单独生成方向和 setup。但每份计划都必须在承担风险前记录并约束：

- 已知日程事件及其时间窗口，例如会影响本品种的经济发布、结算、换月、开闭市变化或预定维护；写明事件前后的禁止新风险、缩短有效期或强制退出边界；
- 行情、时钟、下单、回执、撤单、保护与退出路径是否可用，以及主路径异常时允许使用的备用核对和退出方式；
- 当前品种与持有窗口可预见的尾部风险，例如跳空、流动性骤降、涨跌停或交易暂停、隔夜与结算暴露，以及这些风险如何进入计划 account risk、数量和放弃条件。

外部风险无法量化到可承受边界，或执行环境无法持续核对仓位、维持保护并安全退出时，候选不合格。已有风险如何降级、退出和恢复保护属于执行状态机；策略层只锁定触发这些动作的边界。完整思想边界见[Al Brooks 方法论总纲](../core/00_method/00_al_brooks_thesis.md)。

## 价格、目标与净 R 口径

| 口径 | 回答的问题 | 使用方式 |
| --- | --- | --- |
| 目标区域 | 当前 setup 为什么可能测试这里？ | 来自当时可见的结构、支撑阻力或 magnet，例如旧高低、区间内部目标或量度投射附近；它不承诺价格精确触及某一跳 |
| 页面基线目标 | 这一页统一使用哪个结构参照或公式？ | 为同类交易保持一致；它是仓库选择，不穷尽 Brooks 对该形态的其他合理管理方式 |
| 实际退出价格 | 订单提交在哪个可交易价格，计划回报按哪里计算？ | 必须在目标区域内或沿交易方向的近侧，并按最小跳动、流动性和预先写定的成交余量确定 |

- 本手册所说的“固定”或“冻结”，不是所有交易共用一个价格、点数或 R。它表示当前机会已经根据当时可见的 setup、结构端点和 magnet 得到具体价位；计划形成后不再为迎合后来行情而重画。新的机会必须重新分析并得到自己的入场、止损和目标。
- 具体策略页若写明“唯一目标／固定目标是某价格，并使用全量限价退出”，按该页公式和取整规则得到的可交易价格同时是页面基线目标与实际退出价格，不再临场另选一点。只有页面仅给出目标区域而没有给出唯一价格时，才在计划形成时选择近侧退出价：做多取区域下沿或其入场侧，做空取区域上沿或其入场侧，并记录具体价格。
- 目标区域、页面基线目标和实际退出价格必须来自同一个 setup 与同一组事先可见的结构端点。候选形成以后，不得为提高纸面净 R 而扩大目标区域、改量另一段腿、改选更远 magnet 或使用更有利的区域边缘；也不得把途中一个原本不是本页目标的近端障碍临时改名为目标来挽救计划。页面规定的 premise 失效退出属于持仓管理，不是修改目标。
- 计划形成时必须把目标区域及来源、页面基线目标、实际退出价格、参照边界、计划入场价或保守预计成交价、保护止损价、数量、每跳价值、往返费用和保守滑点写成具体数值。只写“附近”“结构外”或“空间足够”不算计划完整。
- **计划净风险金额**先把保守预计成交价到保护止损的价格距离按最小跳动、每跳价值和数量折算成金额，再加上止损侧预计滑点金额，以及入场和止损退出的全部费用。**计划净回报金额**先把保守预计成交价到实际退出价格的价格距离按同一方法折算成金额，再减去目标侧预计滑点金额，以及入场和目标退出的全部费用；不得使用更远的页面基线目标虚增回报。保守预计成交价已经包含入场侧滑点假设，不重复计算；费用先按对应订单和数量汇总为金额，汇总后不再乘以每跳价值或数量。计划净 R 等于计划净回报金额除以计划净风险金额。存在多层或不同数量时，分别计算每层金额再相加，不能只用未经费用调整的平均价代替。
- Brooks 语境中的“约 2R”通常描述图表上潜在 reward 相对 initial price-action risk 的关系；本手册的计划净 R 还包含费用和保守滑点。两者不能直接等同，更不能把语境性参考改写成固定的成本后净 R 门槛。
- 计划净 R 是结构分析完成后的记录值，不是用来反推价位的预设参数，也不是统一入场门槛。先按当前 context 与 setup 确定入场、结构保护止损、现实目标和实际退出价格，再计算净风险、净回报与净 R；不得为了得到想要的 R 而缩窄止损、扩大目标或更换结构锚点。
- 本手册把“净空间足够”定义为：实际退出位于盈利方向，扣除费用和保守滑点后仍有正净回报，目标在当前路径上合理可达，并且把该 setup 的概率、止损、回报、途中障碍和管理方式放入同一个 Trader's Equation 后仍然有利。净 R 只是这个方程中的风险回报描述，不是胜率估计；较高净 R 不能补足背景、位置或形成顺序，较低净 R 也不能脱离概率单独否定一笔交易。
- 从保守预计成交价到目标区域之间若有事先可见的强支撑、阻力或 magnet，必须记录其具体价格和来源，并按实际退出价格以及该障碍影响后的到达概率重新检查 Trader's Equation。障碍使目标区域不可合理到达或交易者方程不再有利时，入场前放弃；只有策略页本来就把该结构定义为目标时，它才属于目标区域，不能在候选出现后临时改名。

## 等待失效与保护基线

- 未成交订单仍在等待时，价格一旦触及或越过计划保护止损价，入场资格立即失效并结束旧计划，不等待收盘确认，也不把止损移到新极值外。唯一例外是尚未选择方向的 ii 原子二选一计划：母线另一侧同时是相反方向入场触发，按该页的联动规则处理；任一方向成交后，本条立即适用。实际撤单、竞态和仓位核对按[执行手册](../execution/execution_manual.md#七提交撤销和修改订单)处理。
- 策略页规定的未成交有效期同样约束部分成交后的剩余入场数量。部分成交不重置计数，也不延长订单寿命；到期、目标成交、保护止损成交或主动退出流程开始时，所有未成交余量都失去继续增加风险的资格。已成交数量及撤单是否真实完成仍按平台事实核对。
- 页面列明的结构保护止损成交本身表示所选策略依赖的结构已经失效，即 premise 失效。页面规定的收盘型提前退出只是可能更早发生的额外失效路径；仓位确认归零以前，结构保护止损始终有效。
- 结构保护止损与更远的 catastrophe stop 不是同一概念。本手册正式策略页列出的都是结构保护止损。标准人工计划还须在具体页预先写出可观察的 premise-change 条件，例如强反向突破与跟进、外部价格重新获得持续接受或清楚的 Always In 翻转；触发后按主动退出流程处理，仓位归零前结构保护止损继续有效。若为隔离 stop / target 效果而采用“无视觉提前退出”，必须另建 `mechanical_only` 独立研究变体，记录变体 ID 和研究目的；它刻意不实现 Brooks 的动态 premise 管理，不得与标准回放、模拟或实盘直接比较，结果也不得归因为 Brooks 的完整管理。若另设更远 catastrophe stop 作为平台后备保护，也不能被动等待该宽止损成交。概念区别见[Stop Entry 和 Protective Stop](../core/03_acceptance_and_order_logic/00_stop_entry_vs_protective_stop.md)。

## 固定订单基线与标准管理

除非具体页面明确另行规定，本仓库使用一个页面基线目标和一个实际全量退出价格，不分批止盈、不留 runner。Brooks 的目标与 magnet 思想允许按背景选择目标区域，并可能使用部分兑现或 runner；这里的单目标、近侧定价、净 R、取整和等待失效规则，是仓库为一致执行设定的人工订单基线，不应冒充 Brooks 对所有市场和交易者规定的统一参数。标准管理仍包含上一节预写的 premise-change 主动退出；只有独立 `mechanical_only` 研究变体才刻意排除它。

## 策略页与具体 Trade Plan 的接口

每张具体策略页仍须定义该类机会的：

- premise、适用 Context、形成顺序与不适用环境；
- signal bar 或其他参照事件、chart entry trigger、订单方式和计划价格；chart entry bar 在图表触发后追加，不是事前输入；
- 未成交有效期、结构保护价、唯一目标或显式例外、数量与事前风险；
- 本页特有的 premise 削弱、失效、管理分支，以及确有必要的固定时间覆盖。

每一份具体 Trade Plan / 机会实例则必须在候选合格或承担风险前，把 `minimum_remaining_holding_time`、`latest_entry_time`、`forced_exit_start` 实例化为当前品种、周期与 session 的具体时间或 K 线时点。策略页只有在该类机会本来就有固定截止或强制退出要求时才写死或加严这些值；其余页面直接继承本节共同字段，不必复制一组市场特定常数。

这是让计划可执行所需的接口，不是允许重复定义共同规则。Actual / simulated fill bar、平台回执、实际成交、当前仓位、保护状态和异常处置都不是策略页字段；净 R 公式、Actual Risk / MAE 的事后边界，以及固定订单基线与研究变体的共同理由，也不应在家族 README 或具体页另写一套，只链接本页、[K 线角色定义](../core/04_patterns/01_bar_types.md#signal-bar)与执行手册。

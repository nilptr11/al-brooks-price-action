# 趋势反转策略

> **定位：供交易者区分普通修正，以及同一 MTR 过程的两种入场版本**
>
> 本目录只处理成熟趋势失去控制后的逆向机会。趋势中的普通回调仍回到趋势延续策略；只有一根逆向 K 线、一个楔形名称或一次触及均线，都不足以选择这里的策略。

本目录把同一 MTR 过程拆成两个仓库执行版本：**测试失败信号版本**在通道突破、旧极值实际测试失败后，使用第二次反向信号入场；**突破回调信号版本**继续等待强反向突破与跟进，再使用其后第一次完整回调末端的信号入场。这两个名称不是 Brooks 规定的两种 MTR 形态；它们只说明本仓库在同一过程中选择的风险承担时点。“确认”只表示强反向突破获得跟进，不表示反转结果已经确定。

## 怎样选择

| 当时能够确认的过程 | 只选择这一项 |
| --- | --- |
| 原趋势通道已被反向收盘突破，价格实际测试旧极值后失败 | [MTR：旧极值测试失败后的第二次反向信号入场](mtr_old_extreme_test_failure.md) |
| 此前已有通道突破，旧趋势实际测试旧极值或清楚回探后形成有意义的较低高点／较高低点并失败；反方向随后形成强突破和跟进，其后的第一次回调结束 | [MTR：强反向突破与跟进后的第一次回调信号入场](mtr_confirmed_breakout_pullback.md) |
| 成熟趋势出现三次清楚推动，但仍只期待回到三推起点 | [普通楔形后的逆向修正](wedge_correction.md) |
| 紧通道末端出现至少三次加速推进和高潮，只交易获利回吐 | [抛物线高潮后的逆向修正](parabolic_climax_correction.md) |
| 趋势末端最后整理先顺势突破，随后失败并反向触发 | [最终旗形突破失败后的反转](final_flag_reversal.md) |

## 不要把五项策略叠加

普通楔形、抛物线高潮和最终旗形首先是修正计划，不自动等于主要趋势反转。测试失败信号版本必须已经出现通道突破，并实际触及或越过旧极值后测试失败；突破回调信号版本也必须保留通道突破与旧趋势清楚恢复测试的前序，该测试可以实际触及，也可以形成有意义的较低高点／较高低点，再在其后出现强反向突破、跟进和第一次回调。普通强反向突破加第一次回调，不能单独归入 MTR 突破回调信号版本。

同一段反转过程只允许一项活动计划。已经按测试失败信号版本承担风险时，不能又以突破回调信号版本名义增加仓位；已有楔形修正计划时，也不能因为后来出现主要反转解释而临场扩大目标。原计划结束以后，必须从当前价格重新检查新策略的全部条件。

## 重叠时怎样锁定一页

候选选择按区别性事件首次完成的先后决定，不按哪个目标更远或止损更近决定：

1. 通道或主要趋势线已经被反向收盘突破后，旧趋势的恢复测试若实际触及或越过旧极值再失败，可以选择测试失败信号版本；若未触及但清楚回探并形成有意义的较低高点／较高低点，只能继续等待强反向突破、跟进和第一次回调。上述对应过程若先于普通修正页的固定第二次信号完成，就锁定相应 MTR 入场版本。
2. 事先标出的末端旗形先发生顺势突破，并在三根窗口内收回时，最终旗形的区别性事件已经成立；若它早于楔形或高潮页的固定第二次信号，后续反向信号只使用最终旗形页。
3. 没有以上区别性事件时，三推之间有清楚独立回调才使用普通楔形；末段仍是紧通道、回调很小且至少有三次 surge 才使用抛物线高潮。两种结构不能唯一分开时继续等待或放弃。
4. 两个区别性事件同一根完成、当时仍无法说明主要交易 premise 时，不选择限制较少或目标较远的一页；继续等待或放弃。页面一旦锁定，后来出现的新名称不能替换原入场、止损或目标。

测试失败信号版本与突破回调信号版本是同一 MTR 过程的两个进入时点。前一版本出现候选时，要么锁定该页，要么明确拒绝并继续观察；不能保留其订单或仓位，同时等待后一版本增加风险。后一版本只有在强反向突破、跟进和第一次回调全部作为新的当前过程完成，而且原计划已经结束时，才可重新评估。

## 共同纪律

- 开始前标出原趋势通道、最近旧极值、主要回调点和现实目标。
- 逆势计划一律使用止损入场，让价格先朝计划方向触发。
- 每页只使用一个保护位置、一个全量目标，不加仓、不分批退出、不留尾仓。
- 各页目标是本仓库为相应修正或 swing premise 固定的基线价格；Brooks 所说的 two legs 或 TBTL 描述预期路径和管理语境，不自动产生一个唯一价格，也不能被用来临场延长本页目标。
- 目标空间不足、结构边界看不清或合理止损超出个人风险时，直接放弃。
- 多数反转先形成修正或交易区间，不能把一次逆向触发直接写成新趋势。

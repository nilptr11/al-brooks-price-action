# 逆向修正与主要趋势反转策略

> **定位：供交易者区分三类逆向修正，以及同一 MTR 过程的两个风险承担时点**
>
> 本目录包含尚未证明原趋势失控的 wedge、parabolic-climax 与 final-flag 逆向修正，也包含原趋势经过通道破坏和旧极值测试后形成的两个 MTR 版本。趋势中的普通回调仍回到趋势延续策略；只有一根逆向 K 线、一个形态名称或一次触及均线，都不足以选择这里的策略。

本页只在[全局选择状态机](../how_to_choose.md)路由到反转家族后，负责本家族内部分类与重叠优先级；具体资格、触发、参数和特有失效以所选策略页为准。本页不改写时段覆盖、单候选、结束或重入规则。

本目录把同一 MTR 过程拆成两个仓库执行版本：**测试失败信号版本**在通道突破、旧极值实际测试失败后，使用第二次反向信号入场；**突破回调信号版本**继续等待强反向突破与跟进，再使用其后第一次完整回调末端的信号入场。这两个名称不是 Brooks 规定的两种 MTR 形态；它们只说明本仓库在同一过程中选择的风险承担时点。“确认”只表示强反向突破获得跟进，不表示反转结果已经确定。

课程在相近 MTR 语境中常用约 40% 描述强反向突破前的 swing、约 60% 描述突破确认后的版本；这是过程时点的经验交换，不是下面两张仓库精确策略的回测胜率。仓库又增加了实际触及筛选、严格信号、固定订单寿命、唯一目标和固定管理，不能把课程数字直接贴给这些实例。

两种 MTR 版本都要求一条仍然有效、没有中断的证据链。若原趋势在早先的结构破坏以后，以新的持续紧密趋势腿重新建立控制，早先的通道或趋势线破坏、旧极值测试、反向突破与跟进全部过期；后来的 MTR 必须从新结构重新累积破坏、反向压力和旧极值测试，不能跨越这次控制重置拼接证据。

## 怎样选择

下表只决定交易时哪一项可以成为活动计划，不是阅读选单；五张策略页都属于完整理解范围。

| 职责组 | 当时能够确认的过程 | 只选择这一项 |
| --- | --- | --- |
| MTR：较早风险时点 | 原趋势通道已被反向收盘突破，价格实际测试旧极值后失败 | [MTR：旧极值测试失败后的第二次反向信号入场](mtr_old_extreme_test_failure.md) |
| MTR：较晚风险时点 | 此前已有通道突破，旧趋势实际测试旧极值或清楚回探后形成有意义的较低高点／较高低点并失败；反方向随后形成强突破和跟进，其后的第一次回调结束 | [MTR：强反向突破与跟进后的第一次回调信号入场](mtr_confirmed_breakout_pullback.md) |
| 逆向修正 | 成熟趋势出现三次清楚推动，但仍只期待回到三推起点 | [普通楔形后的逆向修正](wedge_correction.md) |
| 逆向修正 | 紧通道末端出现至少三次加速推进，possible climax 出现该页规定的反向过程，只交易获利回吐 | [抛物线式高潮推进后的逆向修正](parabolic_climax_correction.md) |
| 逆向修正 | 趋势末端先标记 candidate final flag，顺势突破随后失败并反向触发 | [最终旗形突破失败后的逆向修正](final_flag_reversal.md) |

## 不要把五项策略叠加

普通楔形、抛物线高潮和最终旗形首先是修正计划，不自动等于主要趋势反转。测试失败信号版本必须已经出现通道突破，并实际触及或越过旧极值后测试失败；突破回调信号版本也必须保留通道突破与旧趋势清楚恢复测试的前序，该测试可以实际触及，也可以形成有意义的较低高点／较高低点，再在其后出现强反向突破、跟进和第一次回调。普通强反向突破加第一次回调，不能单独归入 MTR 突破回调信号版本。

成熟的[扩张三角形](../../core/04_patterns/06_triangles_ii_ioi_oo.md#expanding-triangle)也只提供 MTR 候选背景，不是第六项独立策略，更不会自动触发任一 MTR 版本。它的尺度、交替腿和末端反向压力不能替代当前证据链中的通道破坏、旧趋势恢复测试、测试失败，以及较晚版本所需的强反向突破与跟进。

同一段反转过程只允许一项活动计划。已经按测试失败信号版本承担风险时，不能又以突破回调信号版本名义增加仓位；已有楔形修正计划时，也不能因为后来出现主要反转解释而临场扩大目标。原计划结束以后，必须从当前价格重新检查新策略的全部条件。

## 重叠时怎样锁定一页

候选选择按区别性事件首次完成的先后决定，不按哪个目标更远或止损更近决定：

1. 最近一次控制重置以后，通道或主要趋势线已经被反向收盘突破，旧趋势的恢复测试若实际触及或越过旧极值再失败，可以选择测试失败信号版本；若未触及但清楚回探并形成有意义的较低高点／较高低点，只能继续等待强反向突破、跟进和第一次回调。上述对应过程若先于普通修正页的固定第二次信号完成，就锁定相应 MTR 入场版本。
2. 事先标出的 candidate final flag 先发生顺势突破，并在该策略页规定的窗口内收回时，本页特有的失败事件已经成立；随后仍须出现该页规定的反向信号。失败与反向压力合在一起只构成可交易的 final-flag result evidence，不能在入场前确认它事后确实是 `final`。若失败事件早于楔形或高潮修正页的固定第二次信号，后续反向信号只使用最终旗形页。
3. 没有以上区别性事件时，三推之间有清楚独立回调才使用普通楔形；末段仍是紧通道、回调很小且至少有三次 surge 时，先标记为高潮式推进或 possible climax，待该页反向过程完成后才允许执行修正计划；是否成为已确认 climax 仍由后续市场状态决定。两种结构不能唯一分开时继续等待或放弃。
4. 两个区别性事件同一根完成、当时仍无法说明主要交易 premise 时，不选择限制较少或目标较远的一页；继续等待或放弃。页面一旦锁定，后来出现的新名称不能替换原入场、止损或目标。

测试失败信号版本与突破回调信号版本是同一 MTR 过程的两个进入时点。前一版本出现候选时，要么锁定该页，要么明确拒绝并继续观察；不能保留其订单或仓位，同时等待后一版本增加风险。后一版本只有在强反向突破、跟进和第一次回调全部作为新的当前过程完成，而且原计划已经结束时，才可重新评估。

候选锁定前，只要原趋势形成新的持续紧密趋势腿并重新建立控制，当前 MTR 证据链就立即重置。以后即使再次出现相似形态，也必须从新的通道破坏开始计数；“图上曾经发生过”不等于仍可用于当前计划。

## 反转家族特有约束

价格、净 R、等待失效、保护与固定管理的通用语义只由[策略共同规则](../common_rules.md)定义。下列内容只补充反转家族的选择和目标边界：

各页中的严格 K 线算法、三根形成窗口、两根订单寿命、唯一全量目标和不主动 trailing，都是仓库执行基线；课程支持相应市场过程、结构 stop 与目标候选，但没有规定这套统一精确参数。

- 开始前标出原趋势通道、最近旧极值、主要回调点和现实目标。
- 同时标出最近一次控制重置点；MTR 页使用的每项前序证据都必须位于其后，并属于同一连续过程。
- 逆势计划一律使用止损入场，让价格先朝计划方向触发。
- 各页目标是本仓库为相应修正或 swing premise 固定的基线价格；Brooks 所说的 two legs 或 TBTL 描述预期路径和管理语境，不自动产生一个唯一价格，也不能被用来临场延长本页目标。
- 结构止损与固定目标构成机械订单基线；标准计划中，普通回测或一根失望 K 线不自动退出，但强反向突破与跟进、外部价格持续接受或清楚的 Always In 翻转否定本页 premise 时，仍按[共同 premise-change 规则](../common_rules.md#等待失效与保护基线)主动退出。只测试 stop / target 的回放必须另建 `mechanical_only` 独立研究变体，不得并入标准结果。
- 目标空间不足、结构边界看不清或合理止损使计划 account risk 超出事前上限时，直接放弃。
- 多数反转先形成修正或交易区间，不能把一次逆向触发直接写成新趋势。

共同选择流程见[如何选择并锁定一项策略](../how_to_choose.md)，订单、成交和保护见[执行手册](../../execution/execution_manual.md)。

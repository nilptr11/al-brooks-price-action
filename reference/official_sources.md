| `SRC-TRADING-RANGES` | “In trading range, buy low, sell high, scalp”、“Tight Trading Ranges” | 区间中部应回避；宽区间边缘可以 stop 或 limit fade；紧区间多数交易者应等待，除非范围足以支持可盈利的 scalp。 |

| `SRC-GLOSSARY` | `gap`、`gap reversal`、`body gap`、`breakout` | Gap 可以是 K 线价格或均线等支撑阻力对象之间的空间；opening gap 由当日第一根开盘越过昨日最后一根或整个昨日范围形成；gap reversal 的最低几何事件是当前 K 线向 gap 方向越过前一根至少一跳，并不要求已经到达真正的 gap 边界；body gap 的官方 5—10 根描述是回测案例，实体不重叠才是核心关系。 |

| `SRC-STRONG-LEGS-2016` | slides 8–14 | 官方实务连续谱是 breakout、tight channel、broad channel、trading range；各结构可只持续 1–2 根或 20 根以上。随 pullback 加深、重叠增加和 gap 减少，双边获利能力逐步提高，而不是在固定阈值处突然切换。 |
| `SRC-STRONG-LEGS-2016` | slides 5–7 | Gap 可以是任意支撑阻力对象之间的空间；趋势中段的 gap 可使用中点或特定边缘形成 measuring-gap 候选，成熟趋势末端最强突破随后被反向收回时则支持 exhaustion 解释。 |
| `SRC-STRONG-LEGS-2016` | slides 45–48 | 宽区间的 limit 方案可能需要远至重要 swing / measured-move 区域的 stop，并按完整风险缩仓；区间内存在多个合理兑现位置。 |
| `SRC-GLOSSARY` | `breakout`、`breakout mode`、`follow-through`、`major trend reversal` | Breakout 的最低定义是高低点越过重要旧价位；强收盘和 follow-through 是质量证据，回踩不是最低定义中的必要事件；breakout mode 表示任一方向突破都应有获得跟进的可能；完整 MTR 包含通道突破和对旧趋势极值的测试。 |
| `SRC-GLOSSARY` | `climax`、`spike and channel`、`tight channel`、`minor trend reversal`、`reversal` | Climax 的最低定义包含过快过远之后已经转向 trading range 或反向 trend，且多数只转成 trading range；channel 是 breakout 后动能较低、双边交易增加的 trend 形态；minor reversal 通常先成为 pullback，扩展约 20 根后才可能演化成 trading range。 |
| `SRC-GLOSSARY` | `measured move in bull/bear trend`、`measuring gap`、`micro measuring gap` | Measured move 的最低例子是当前腿约等于更早同向腿，或约等于此前交易区间/形态高度；measuring gap 是最终带来 measured move 的 breakout；micro measuring gap 是强趋势 K 线前后完整区间不重叠的强度/量度线索，两条边界的中点是官方量度参考。 |
| `SRC-GLOSSARY` | `exhaustion gap in bull trend`、`exhaustion gap in bear trend` | Exhaustion gap 是趋势后期异常大的顺势 K 线或连续 K 线之后出现至少小反转；缺口参考区从末端收盘到 breakout point，也可延伸到 climax bars 的另一端。官方最低定义不要求快速或完整回补。 |
| `SRC-GLOSSARY` | `setup`、`signal bar`、`entry bar`、`high 1, 2, 3, or 4`、`low 1, 2, 3, or 4`、`second entry`、`second signal` | Setup 的官方最低定义是带 context、可作为入场单依据的 pattern，不等于完整风险计划；实际 fill 后最后一根才成为 signal bar，fill 所在 K 线是 entry bar；H/L 计数要求严格越过前高/前低；second entry 与 second signal 由是否实际形成 entry bar 区分。 |
| `SRC-GLOSSARY` | `micro double bottom`、`micro double top`、`oo` | Micro double bottom 在 bear spike 中可是一根 one-bar bear flag，在 bull flag 末端可成为向上 reversal setup；micro double top 在 bull spike 中可是一根 one-bar bull flag，在 bear flag 末端可成为向下 reversal setup。OO 是一根 outside bar 后紧接一根更大的 outside bar。 |
| `SRC-GLOSSARY` | `failure (a failed move)`、`trap`、`trapped in a trade`、`success` | Failure 的结果边界是尚未先获得 scalper's profit 或原目标而 protective stop 先到，通常随后因入场者退出推动反向运动；trapped in a trade 要求实际持仓处于开放亏损且尚未获得 scalper's profit。由这些定义可知，弱 entry bar 或没有立即 follow-through 只是弱化证据，不能单独确认 failure 或 trapped traders。Success 要求目标先于 protective stop 到达。 |
| `SRC-GLOSSARY` | `micro channel`、`pause bar`、`pullback bar`、`pullback`、`leg`、`swing`、`trading range` | Micro channel 可有零次或一两次小回调，单调低点/高点只是严格子型；pause bar 是一种 pullback，与 bar pullback 的几何条件可以重叠；leg / swing 都是较小趋势结构；trading range 的最低定义可以从单根高度重叠 K 线开始，但不自动构成可交易区间。 |
| `SRC-GLOSSARY` | `breakout test`、`disappointment bar`、`failed failure`、`risky` | Breakout test 可在一两根内或延后 20 根以上，不含“通常 5–10 根”的要求；Disappointment Bar 在 glossary 中是 BTC/STC 趋势里的窄义角色；failed failure 恢复原突破方向并成为 breakout pullback / second signal，官方称其更可靠，但名称不保证运动幅度。 |
| `SRC-MAKE-MONEY-2016` | “Small risk, big reward”、“Use correct stop and targets” | Ask Al 明确把较大的 initial risk 与可能较小的 actual risk 对照，并强调 correct stop、目标和小仓位；它没有在公开文本中给出足以支持仓位公式的统一 Actual Risk 算法。 |
| `SRC-BREAKOUTS-2025` | slides 4–5、11、16 | 强 breakout 同时带来较高概率、较大结构 stop 和 measured-move 目标；顺势方可把 stop 放在完整 breakout leg 外，之后在新强突破形成 major higher low / lower high 时保护性 trailing。 |
| `SRC-LIVE-TR-BO-2021` | “Introduction”、“Trade entry limit order filled”、“Trade and stop management”、“Profit targets” | Gap down 远离均线并未自动形成做空结论，连续大牛 K 线改变了控制读取；Al 随后在区间突破案例中比较完整 breakout stop、较近管理 stop、两个 measured-move 投射和昨日低点阻力，并按目标区近侧的现实利润挂单。 |
| `SRC-GOOD-TRADE-2017` | slides 5–10 | 每笔交易应有活动 protective stop；合理反向信号和强反向 breakout 可以成为提前退出依据。图上经常存在两三个合理 stop，较远 stop 要配更小仓位。 |

## 使用规则

- 引用时使用 Source ID，并尽量补充页面条目、章节、页码或 slide。
- “来源可访问”不等于“结论已核对”；结论状态以 `audit_matrix.md` 为准。
- 直接转述、仓库推导和保守执行政策必须分开标记。
- 若官方资料之间存在差异，记录各自语境，不自动用较新的页面覆盖较早资料。

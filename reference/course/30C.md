# 30C：Z-score 材料冲突、概率优先与等距目标判断

来源：`SRC-COURSE-01-36/30C.txt`，对应 PDF 物理页 2344–2367。

## 内容结构

1. 给出基于胜负连续段数的 Z-score 公式，并把它称为系统可靠性指标（PDF p2345–2348）。
2. 比较“多数利润来自单笔”的低 Z 系统与风险控制、初学者小仓位（PDF p2349–2354）。
3. 强调概率比 RR 更难判断，并给出 90% 区间/宽通道、10% 强突破/紧通道的经验划分（PDF p2355–2357）。
4. 定义等距和不同距离的先到达概率，并用 Always In 表示等距优势方向（PDF p2358–2361）。
5. 比较 80% 高概率/3:1 不利风险回报与 60%/1R 目标（PDF p2362–2365）。

## 核心思想

- 本讲给出的 Z-score 使用总交易数 `N`、盈利/亏损连续段数 `R`、赢家数 `W`、输家数 `L` 和 `X=2WL`，意图衡量胜负序列的非随机性/回测可靠性，而非单笔价格形态（PDF p2346）。
- 课程声称高 Z 系统胜率高、赢家规模相近，过去结果更能预测未来，即使错过部分交易也较可靠；例子是 H2 多头旗形/L2 空头旗形第二次入场短线，十二笔中十一笔盈利（PDF p2347）。
- 课程又把高 Z 与较低破产风险联系，声称罕见最佳值约 5、优秀交易者常在 2–3，并可把每笔账户风险提高到约 2%（PDF p2348）。
- 相反，低 Z 系统被描述为胜率低、依靠极少数大赢家、结果方差大；若错过关键赢家就整体亏损，且连续许多亏损在心理上难以执行（PDF p2349–2351）。
- 无论系统评价如何，初学者都应小仓位。课程建议每笔账户风险低于约 1%，承认学习阶段会因执行错误亏损，把损失限制为学习成本（PDF p2352）。
- “不在乎仓位”指金额小到不会干扰决策：一万美元 Emini 账户，2 点止损、1 手约风险 100 美元；EURUSD 可用一万单位和 20 pip 止损，把风险降到约 20 美元（PDF p2353–2354）。
- 课程称概率是最重要、也最模糊的变量。初学者常等到趋势显而易见才追入，回调出现又误认为趋势反转，因恐惧和不确定而错过或过早退出（PDF p2355–2356）。
- 课程给出经验划分：约 90% 的 K 线位于交易区间或宽幅通道，可双向交易但依赖管理；约 10% 位于强突破或紧密通道，应只顺势（PDF p2357）。
- 概率问题可统一为“哪一侧先到达”：从当前价起，向上 X 单位与向下 X 单位哪个先发生；单位可为 tick、pip、金额或百分比（PDF p2358）。
- 当多空风险和目标距离相等时，Always In Short 表示等距先到达概率偏空，因此卖出比买入更有利；Always In Long 完全镜像（PDF p2359）。
- 实际目标常不等距，系统需要估计向一侧 X 单位先于另一侧 Y 单位的概率，并在概率较高时寻找使交易者方程式仍为正的更优目标（PDF p2360–2361）。
- 课程不要求人工交易者计算完美最优值，只需选择逻辑止损和足够好的目标。高概率时，目标不必固定在 1R，可尝试略高于 1R，但仍需保持正期望（PDF p2361）。
- “很可能达到”在课程词汇中至少约 60%。若目标与风险都是 1R，60% 胜率的毛期望为 `0.6R−0.4R=0.2R`，可以作为高概率短线（PDF p2363–2365）。
- 每一个可见支撑阻力都能转成先到达问题：计算目标距离 X，再问价格到目标前是否更可能先向反方向移动同样 X 并触发结构止损（PDF p2363）。

## 重复与递进

- p2346–2351 从 Z-score 公式、十二笔高胜率例子、破产风险，到单一大赢家系统，试图建立“稳定小赢家优于低胜率大赢家”的教学对比。
- p2352–2354 从账户百分比风险递进到 Emini/外汇具体仓位，是风险控制例子而非 Z-score 数学的必然推论。
- p2356–2357 从概率造成心理错误递进到 90/10 市场环境分类，连接前面交易区间与紧密通道章节。
- p2358–2361 从等距移动、Always In 方向优势，扩展到不同距离和算法目标优化，建立概率的操作性定义。
- p2362–2365 先展示极高概率但很差 RR，再收敛到课程较推荐的“至少 60%、目标 1R”，与 30A–30B 的交易者方程式重复。
- “概率最重要”并非新增第四变量，而是强调它最不确定、最常被忽略；数学上仍必须与回报和风险共同使用。

## 可能矛盾或边界

- 本讲 Z-score 公式只使用胜负顺序、赢家/输家数量和连续段数，不包含每笔盈亏金额。p2349–2351 却用“多数利润来自一笔大赢家”“结果标准差大”解释 Z-score 高低，属于公式与解释直接不匹配。
- p2347 声称高 Z 是因为十一胜一负且赢家大小相同，但给出的公式不使用赢家金额；高胜率本身也不必然产生高绝对 Z，仍取决于胜负连续段排列。这是另一处内部数学冲突。
- 十二笔中十一胜的样本太小，不能据此断言过去结果是未来的良好预测；还存在选择样本、过拟合和市场状态改变。
- Z-score 与破产概率不是一一映射。破产风险还取决于净期望、盈亏幅度、尾部、相关性、杠杆和每笔仓位；仅凭 Z=2–3 不能推出每笔可风险 2%。
- “最佳 Z 约 5、优秀交易者 2–3”是具体统计主张，材料没有定义正负方向、样本规模或来源，不能作为可靠阈值。
- 低胜率、少数大赢家的趋势跟随系统仍可能正期望且稳健；它不因依赖大赢家就自动是坏系统。真正问题是执行遗漏、方差、仓位和样本稳健性。
- 初学者每笔低于 1%、熟练者 2% 是经验风险上限，不适用于所有波动、相关持仓或连败长度；组合同时风险比单笔百分比更重要。
- Emini 2 点×1 手约 100 美元未计手续费和滑点；外汇 pip 金额还会随报价货币、账户币种和汇率变化，例子只是近似。
- “概率最重要”与交易者方程式三变量并不意味着可忽略 RR。p2362 恰好说明极高概率配极差 RR 仍可能不适合交易。
- 90%/10% 的市场状态比例是无样本的经验主张，且“任何图表”过宽；不同品种、周期和波动环境可能不同。
- “摆脱情绪、遵循计算机逻辑才会赚钱”是绝对化教学措辞。情绪管理重要，但计算机策略也会失败，人工判断不可能完全无情绪。
- p2362 给出概率约 80%、风险约为回报 3 倍。按本课程自己的方程式，毛期望为 `0.8×1−0.2×3=+0.2R`，却又笼统说“风险大于回报会亏钱”，构成直接数学矛盾；若考虑成本或概率估计误差，需明确写出，原页没有。
- 分批加仓会让 p2362 的实际亏损超过单一 3R，可能解释“不适合大多数人”，但仍不能替代精确总风险计算。
- “很可能至少 60%”是课程词汇约定，不是概率学定义。实际 55% 配 2R 也可能正期望，60% 不是交易者方程式的一般门槛。
- Always In 表示课程对当前等距方向概率的判断，不保证任何给定 X 距离都保持同一优势；更远目标会穿越不同支撑阻力，概率需持续更新。
- RR 的最优目标取决于完整分布和路径，人工只用等距比较是简化，不等同于算法真正优化。

## 翻译或抽取问题

- PDF p2346 公式中的 `.S` 很可能是 `0.5` 的 OCR 错误，指数 `A(l/2)` 应为平方根/二分之一次幂；当前文本不宜直接复制为可计算公式。
- PDF p2346 把 Z-score 括注为“离均值的标准差数”，但此页公式是胜负 runs 统计量，不是收益序列普通 Z 分数；现有材料不足以说明两种术语来源的关系。
- PDF p2347 的十二笔交易、胜负排列和赢家大小需图形；即使可见，也不足以验证未来预测力。
- PDF p2348 的 Z=5、2–3 和 2% 风险没有方法说明，不能由本页公式推导。
- PDF p2353 的 Emini 每点合约价值未在幻灯片文字中说明，100 美元由合约规格隐含，且未作外部核验。
- PDF p2354 中文把 pip 译为“点”，应保留 pip，避免与 Emini point 混同。
- PDF p2358 英文把 `Profit target (risk) for the bears` 写成 risk，按对称结构应为 reward；中文沿用了“也是他们的风险”，属于原页/翻译概念错误。
- PDF p2360 英文是“估计先上涨 0.1%、还是先下跌 0.13% 的概率”，中文误译成“该概率为 0.1%”，把价格幅度当成概率值。
- PDF p2362 的 80%、3 倍风险、弱反转和加仓点依赖图示；不影响其与方程式的算术冲突。
- PDF p2365 的 `with stop below PB` 很可能指用卖出止损单在回调下方入场，不是把空头保护止损设在回调低点下方；中文误译成保护止损会导致方向错误。
- PDF p2344–2345、p2355、p2366–2367 为封面、目录、分节或结束页，不提供新增独立论据。

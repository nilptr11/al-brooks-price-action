# 13C：Always In 下的 Scalping 与 Swing

- 来源：`阿布价格行为学_01-36.pdf`
- 物理页码：855–868
- 课程位置：第 13 讲 C

## 内容结构

1. 说明 AIS/AIL 中顺势 scalp 的常见入场和退出方式。
2. 比较等待清晰 Always In 反转与提前交易次要反转两种风格。
3. 用同一多头行情比较多次 scalp 与一次 swing 的执行负担和结果。
4. 说明 Always In 方向依赖时间周期。
5. 补充 AIS swing 持有到方向真正切换的管理方法。

## 核心思想

### 1. 顺势 scalp 仍以已确认的 Always In 为前提

AIS 中，顺势 scalper 可以在回调后再次向下反转时卖出，在卖出信号 K 线低点下方使用 stop entry，依赖保护性止损，并以小额利润退出（约 p857–858）。AIL 情况镜像。

“因任何理由卖出”只属于已确认 AIS 的强趋势环境，不应脱离状态背景推广到交易区间或方向未明的行情。

### 2. 多次 scalp 的理论机会不等于可稳定全部执行

课程图例中，如果交易者抓住四次空头 scalp 且每次都正确管理，可以得到四笔小利润，但页面明确说这种完整执行并不太可能（约 p858）。

机会数量增加也增加判断、入场、退出和重入的错误概率，不能只汇总事后最优交易。

### 3. 保守交易者等待清晰方向切换

Always In 反转经常不清楚。部分交易者只在反转已经明确发生后才翻向，因此交易次数和决策更少、压力更低，且单次反转成功概率更高；在新的清晰反转出现前，继续沿上一次清晰方向交易（约 p859）。

代价是入场更晚，也会错过部分早期反转利润。

### 4. 更主动的交易者可以尝试次要反转

愿意承担更多交易和决策的交易者，可以在楔形底、强买入信号等次要反转处尝试转多，以换取更多机会。但次要反转本身不是高概率交易（约 p860）。

若整体仍是强空头通道，三角形给出高概率 AIS，且没有清晰底部，未持空者仍应沿原方向卖出。主动交易并不等于忽略主导状态。

### 5. 强突破建立 AIL 后，可用回调棒高点买入

一根大阳线收盘远高于过去二十根 K 线高点，可以建立 AIL。此后顺势买入；最简单的回调入场方式，是把 buy stop 放在回调 K 线高点上方，等待价格恢复上涨触发（约 p861–862）。

这一执行依赖前置强突破已经足以建立 AIL，不能把任意回调棒高点突破视为同等质量。

### 6. 多次多头 scalp 的案例收益依赖完美执行

同一图例中，五次多头 scalp 若全部参与并正确管理，其中三笔各约盈利二十 pips、两笔保本，总计约六十 pips。课程同时强调，很难同时完成五次交易并五次都做出正确决定（约 p863）。

这些数字只描述单个事后案例，不是策略的预期收益或保证。

### 7. 一次 swing 在该案例中更容易且结果更好

同一行情中，swing trader 在首次买入后持续持有，直到楔形更高高点主要趋势反转、最终旗形、强卖出信号和微型双顶等顶部证据出现，约获得七十八 pips。课程认为该次执行比五次 scalp 更容易、更确定（约 p864）。

这是同图案例比较，说明减少决策次数可能提高可执行性，不证明所有 swing 在所有行情中都优于 scalp。

### 8. Always In 必须附带时间周期

六十分钟图可以仍为 AIL、策略是买回调，同时五分钟图已经出现楔形顶并转向 AIS，空头可以做 scalp 或 swing（约 p865）。

多周期方向不同不是逻辑矛盾，而是不同尺度的状态。任何 Always In 结论都应记录时间周期。

### 9. AIS swing 可以持有到方向真正转为 AIL

AIS swing trader 可以在首个卖出信号 K 线低点下方入场，只要市场没有转为 AIL 就继续持有。图例持有到日末，收益与多次 scalper 接近，但执行更简单、更确定（约 p867）。

这里的“没有理由提前退出”只针对该图例中方向未切换的前提；仍需服从保护性止损和交易时段管理。

- p857–858 先给 AIS 中“顺势卖”的原则，再补具体 stop entry、止损和 scalp 退出，属于由原则到执行。
- p859 与 p860 形成交易风格对照：等待清晰反转 versus 尝试次要反转。两页不是互相否定，而是决策频率和入场时机不同。
- p861–863 使用同一多头案例，从建立 AIL、回调入场到五次 scalp 结果，属于连续展开。

- p867 在空头案例中再次比较一次 swing 与多次 scalp，核心原则与 p864 重复，但增加了“只要仍 AIS 就持有”的明确退出条件。
- 13C 重复 13A/13B 的“因任何理由顺势”和“方向切换需确认”，本讲新增的是 scalp/swing 的执行复杂度比较。

## 可能矛盾或边界

1. **等待清晰反转与交易次要反转都是有效风格**：前者更少交易、较晚确认，后者更多机会、较低单次概率和更高决策负担。
2. **次要反转可交易，不等于它已改变 Always In**：逆势交易通常先按 scalp 管理，除非后续真正建立新方向。
3. **“因任何理由买卖”只适用于确认后的强 Always In**：它不是对信号质量和风险管理的普遍豁免。
4. **案例中 swing 收益高于 scalp 不是通用排序**：差异包含事后挑选、执行次数和具体行情路径。
5. **不同时间周期可同时给出相反方向**：策略和止损必须与所交易的周期一致，不能用六十分钟 AIL 否定五分钟 AIS，也不能反过来。
6. **持有到日末是案例管理，不是 swing 定义**：真正原则是持有到方向切换、目标、止损或其他管理条件触发。

## 翻译或抽取问题

- p855、p868 的片头片尾和价格轴有 OCR 噪声。
- p857 中文标题“无论无何”是文字错误；更重要的是 `sell for any reason` 不能脱离 AIS 限定。
- p858 的 `sell stop` 是卖出止损入场单，不是保护性止损；同页另有 `rely on stop` 指保护性止损，中文若都简写“止损单”会混淆职责。
- p859 多组图注在线性抽取后顺序混合，楔形底/HL MTR 与前面的强空头突破分别属于不同状态节点。
- p860 双顶空头旗形、楔形底买入和强空头三角形卖出三个场景被抽成一段，不能当作同一时点同时执行。
- p863 中文把 `pips` 译为“点”，后续精确核对时应统一 pips、ticks 与点值，当前只保留原单位。
- p864 的“更确定”是课程对该案例执行难度的评价，不应改写为 swing 具有确定盈利。
- p865 同页含六十分钟与五分钟两个状态，抽取时若漏掉周期标签会形成 AIL/AIS 直接矛盾。
- p867 的幻灯片编号为 `Slide 4`，但物理顺序位于 `Slide 12` 之后，属于来源编排或编号异常；内容仍属于本讲，但应保留顺序异常记录。

# 03C：Pip 价值、报价界面与现货/期货比较

来源：`阿布价格行为学_01-36.pdf` p204-231

## 内容结构

- p204-210：仓位、货币对和汇率如何决定 pip 价值。
- p211-214：历史外汇工作区、FXTrader 和价格梯界面。
- p215-221：成交对手方、bid/ask、spread 及立即往返成本。
- p222-229：现货外汇与 Euro FX futures 的规模、保证金和成本比较。
- p230-231：回顾与结束。

## 核心思想

- pip 价值由仓位、pip 大小和报价货币共同决定。课程示例中，10,000 单位约
  $1/pip、100,000 EURUSD 约 $10/pip；USDJPY 的 pip 先以日元计算，再按汇率
  换成美元（p206-210）。
- 平台页面展示了课程年代的外汇工作区、价格梯和五分钟图，主要是操作环境示例，
  没有新增价格行为原则（p212-214）。
- 成交必须有相对方。课程用做市商式模型说明经纪商在某个价格提供另一边，但
  实际经纪商可能采用做市、代理或混合执行模式（p215）。
- 买入基础货币支付较高 ask，卖出基础货币收到较低 bid，差额为 spread。立即
  买入再卖出会损失一个价差（p216-221）。
- 现货外汇可以使用很小仓位；期货有标准合约、集中成交和保证金要求。课程用
  当时 EURUSD/EC 的图表、名义额和成本作对比（p223-228）。
- 较少交易的货币对通常点差更宽，但点差的美元成本仍需按报价货币、仓位和汇率
  换算，不能只看 pips（p229）。

## 重复与递进

- p206-210：从一般仓位规则、EURUSD 直接美元计价，到 USDJPY 日元计价和美元
  换算，每页增加一步，不宜压成一句“100,000 单位固定 $10/pip”。
- p218-221：依次讲价差收入模型、两边报价、1.4 pips/$14 算术和立即往返结果，
  是机制—算例—后果的链条。
- p223、p225、p227：从现货/期货定性总览，推进到名义额/规格，再到成本比较。
- p224、p226、p228 是前一英文页的中文拆分页，属于翻译重复，不是新的 Brooks
  思想。
- p205、p211、p222、p230 是导航和回顾。

## 可能矛盾或边界

- p215 的 `Market` 指能让买卖双方成交的市场，不是“市价单”；也不能由本页推出
  所有经纪商总是最终风险对手方。
- p218-221 的 spread 模型不能被理解成一次往返支付两次完整价差，也不能推出
  spread 全部等于经纪商净利润。
- p219 标题仍是 1.3161/1.3162，正文报价已换成 1.11347/1.11361，同页版本未
  同步；实际 1.4-pip 算术应以正文为准。
- p223 的一组现货/期货图近似，只能说明该示例，不能证明所有市场、所有时期的
  图形或执行条件都等价。
- p225、p227 的合约规模、tick、spread、commission 和账户门槛都是历史参数，
  不构成永久成本排序。
- p229 报价差约 11 pips；按本讲自身公式，100,000 单位对应 110 NZD，无法直接
  得到正文的 `$7 each way`。这是材料内部尚未解释的数字矛盾。

## 翻译或抽取问题

- p210：英文是 `0.0000959` 美元/单位，中文写成 `0.00000959`，少一数量级，
  但仍给出 $9.59 结果。
- p213：raw text 几乎只剩页脚，完整平台界面没有被提取；纯文本分析只能记录
  页面用途，不能恢复具体按钮和列。
- p217：中文把客户在 ask 买、bid 卖的动作反转，并把 Ask/Bid 简化为做空/做多，
  与英文定义不一致。
- p219：标题报价和正文报价不一致，属于课件内部版本残留。
- p224、p226、p228 只有译者复述，且没有被原候选定位关联到前一英文页。
- p226：把 `$155,000` 写成“155.5 万美元”，并给 tick 数错误加上美元单位。
- p229：截图中的下单数量并不统一，进一步削弱 `$7 each way` 的可复算性。
- 多页 extracted title 漏掉蓝色标题；p213 是本讲最明显的整页文本抽取缺失。

# 32A：Stop、Limit、Bracket/OCO 与 Market 订单

来源：`SRC-COURSE-01-36/32A.txt`，对应 PDF 物理页 2500–2526。

## 内容结构

1. 列出 stop、limit、bracket/OCO、market 四类常用订单，并区分入场 stop 与保护性止损（PDF p2501–2503）。
2. 讨论黑天鹅、系统失效、跳空和滑点，说明保护止损只能限定常态风险，不能保证最终成交价（PDF p2504–2509）。
3. 说明限价单用于止盈或逆势入场，以及“触价”和“穿价”对成交确定性的影响（PDF p2510–2512）。
4. 解释 bracket/OCO 如何在入场成交后同时挂出止盈限价和保护止损，并在一边成交后取消另一边（PDF p2513–2516）。
5. 介绍市价单的快速进出、自动取消残余订单、外汇图表价格与实际成交差异，以及经纪商成交争议（PDF p2517–2526）。

## 核心思想

- 课程把大多数人工交易所需订单归为四类：stop order、limit order、bracket/OCO order 和 market order。它们不是四种交易策略，而是四种触发、价格约束和联动方式（PDF p2501–2502）。
- 卖出 stop 放在当前价下方，买入 stop 放在当前价上方，既可用于突破入场，也可用于退出已有仓位。入场 stop 与保护性止损都使用 stop 机制，但交易目的不同（PDF p2502–2503）。
- 课程建议多数交易者主要用 stop order 顺着价格突破入场；任何仓位一旦建立，应立即设置保护性止损，防止犹豫、不愿认错或意外行情把可控损失拖成账户级损失（PDF p2502–2503）。
- 空头保护止损位于当前价/结构高点上方，是买入退出；多头保护止损位于当前价/结构低点下方，是卖出退出。方向应按“平掉现有仓位”理解，不能只看 stop 的多空名称（PDF p2503）。
- 保护止损只规定触发价，不保证成交在该价。市场跳空、流动性消失、交易所或经纪系统过载时，stop 触发后的第一笔可成交价格可能远离计划价（PDF p2504–2509）。
- 股票财报后跳空案例中，22.56 买入、21.99 止损的名义风险是 0.57，但次日首个可成交价约 18.00，实际损失约 4.56；因此计划风险只是常态下的最低/预期风险（PDF p2506–2507）。
- USDCHF 案例同样用极端隔夜跳空说明：1.0190 买入、1.0136 止损名义风险 54 pips，实际开盘成交远低于止损，损失超过一千 pips（PDF p2508–2509）。
- 课程提出买入 put 可限制持股下行尾部风险，但需要支付权利金；这说明止损和期权保护解决的是不同风险：止损依赖市场成交，期权在约定期限和执行价提供权利（PDF p2507）。
- 限价单用于“只接受该价或更优价格”的入场/退出。多头可把卖出限价放在前高止盈，空头也可在前高挂卖出限价赌突破失败；前低处的买入限价完全镜像（PDF p2511–2512）。
- 价格只触及限价位时，订单可能因队列和成交量未成交；课程采取保守经验，认为价格通常需要穿过一个最小单位，才能较有把握确认限价单已成交（PDF p2511–2512）。
- bracket order 是一组退出订单：止盈限价单加保护性止损单，通常使用 OCO 关系。入场成交后两边同时生效，其中一边成交，另一边应自动取消（PDF p2514）。
- 止盈和止损距入场点不必相等，也可放置多个限价单分批止盈。交易软件可在提交入场时预先创建 bracket，待入场真正成交后再把退出订单发送/激活（PDF p2514）。
- 空头 swing 案例用前一 K 线下方卖出 stop 入场，止盈远低于入场、保护止损在趋势高点上方；止损先成交后，OCO 取消未成交止盈单。多头 scalp 案例则在前高限价止盈，成交后取消下方保护止损（PDF p2515–2516）。
- 市价单优先立即获得成交，而不是等待突破 stop 或回调 limit。买入通常从卖价/offer 一侧成交，卖出通常从买价/bid 一侧成交，实际均价还取决于可用深度（PDF p2518）。
- 市价单适合需要马上确认强信号或马上否定交易前提的情形：例如强多头 K 线形成中即时买入，或连续强空头 K 线使原多头 scalp 失效时立即退出（PDF p2519–2520）。
- 手动市价退出后必须确认原 bracket 已取消，否则残留止盈或止损可能随后触发并意外建立反向仓位；不能完全假设所有软件都会正确自动处理（PDF p2520–2521）。
- 许多零售外汇图表基于 bid 或 ask 报价而非集中交易所实际成交，因此图上看似触及订单价格，不等于经纪商的正确触发侧或真实可成交价已满足（PDF p2522）。
- 对成交或未成交有疑问时应保存订单记录并联系经纪商。课程强调先检查自己的订单参数和平台状态，但其“Not Held/责任”说法是经纪合同层面的概括（PDF p2524）。

## 重复与递进

- p2502–2503 先给 stop 的机械定义，再从行为风险解释保护止损为何最重要，属于从订单机制到交易纪律的递进。
- p2504–2509 用黑天鹅概念、股票跳空和 USDCHF 跳空反复说明“止损价不等于成交价”。三个层次分别是系统性风险、常见股票事件和极端外汇事件，保留有教学价值。
- p2511–2512 是多头止盈/空头逆势卖出与空头止盈/多头逆势买入的方向镜像，规则相同，可在仓库中用对照表合并。
- p2514 给出 OCO 一般机制，p2515–2516 用空头 swing 和多头 scalp 演示哪一边成交后取消另一边，是合理实例化。
- p2518–2521 反复说明市价入场/退出后软件创建或取消 bracket，真正新增的是“人工退出后检查残余订单”。
- p2522–2524 从特殊订单类型递进到外汇报价图和成交争议，属于执行基础设施边界，不应与价格行为概率混为一谈。

## 可能矛盾或边界

- p2502 说市场到达 stop 价格就会成交，后面 p2506–2509 才说明可能在远差于 stop 的价格成交。准确表述应是 stop 被触发后转成可执行订单，成交价不受 stop 价保证。
- “始终设置保护性止损”是必要的账户纪律，但不能独自防止跳空、停牌、系统中断和流动性枯竭；还需小仓位、隔夜风险限制、分散和必要时的期权保护。
- “所有经纪软件和交易所大约每十年都会失败一次”是无定义、无样本的绝对经验主张；不同故障类型和机构不能合并成统一频率。
- 1987 一天跌 25%、2010 36 分钟跌 9%、1929 三年跌 90% 的事件描述使用不同指数、窗口和四舍五入口径，不能作为彼此可比的统计样本；“1929 Black Friday”名称也需后续来源核对。
- 黑天鹅按概念通常强调事前难以纳入模型，材料又把已知历史崩盘统称黑天鹅并赋予低概率；这是宽泛教学用法，不是严格风险学定义。
- “大多数机构买 puts”是具体市场行为主张，材料未说明机构类型、资产、期限或数据；put 保护还存在权利金、到期、执行价和基差问题，现有材料不足以支持普遍结论。
- p2507 的 put 只能在其覆盖标的、数量、执行价和到期日内提供保护，不等于任何情况下都比 stop 更好。
- “90% 的限价单需要价格穿过才成交”没有订单队列和市场样本支持。触价即可全部成交、部分成交或不成交；穿价也会受停牌、撤单、断线等影响。
- bracket/OCO 的创建、发送、托管和取消可能发生在本地软件、经纪服务器或交易所。若只在本地维护，平台断线会破坏联动；材料把所有软件行为表述成统一机制过于简化。
- p2516 英文称多头入场时 `as soon as his sell order is filled`，方向错误；应是 buy entry order 成交后挂出 bracket。
- 市价单并不保证“立即全部以当前 offer/bid 成交”。流动性不足、快速行情、停牌和订单过大时会出现滑点、分笔、部分成交或拒单。
- “stop-limit 和 marketable-limit 现在最好都避免”过度泛化。前者可限制最差成交价但承担不成交风险，后者可限制异常滑点；是否适合取决于流动性和执行目标。
- 外汇市场并非所有图表都只基于单侧报价；数据源、经纪商和产品不同。订单触发使用 bid、ask、last 或其他规则也由具体合同决定。
- p2524 的“99% 是交易者错误、1% 是经纪商/交易所错误”没有统计依据；“Not Held”条款的责任范围也因司法辖区、产品和合同而异，不能当作普遍法律结论。
- “即使不是你的错也永远是你的责任”适合作为操作风险纪律，不应误写成放弃保留日志、申诉、监管投诉或合同权利。

## 翻译或抽取问题

- PDF p2501–2503 将 stop order、sell stop、buy stop 和 protective stop 都译成“止损订单/止损多单/止损空单”，严重混淆。应分别译为“停止单（触发单）”“卖出停止单”“买入停止单”“保护性止损单”。
- PDF p2501、p2513–2516 将 bracket order 译为“一篮子订单”不准确，应统一为“括号订单/止盈止损组合单（OCO）”；basket order 是另一概念。
- PDF p2504 的历史事件名称和幅度属于具体事实主张；这里只记录课程陈述，不用抽取文本自行修正。
- PDF p2507 中文“做多看跌期权”可简化为“买入看跌期权”，避免把期权方向与标的多头混淆。
- PDF p2509 英文抽取的开盘成交价为 `88.94`，图轴却在 0.90–1.00 附近；应有丢失小数点或数字顺序问题。该值与“损失 1,242 pips”也无法直接一致复算，不能引用为精确价格。
- PDF p2511–2512 英文是 `go through the limit order price`，中文却译为“达到该价格”，遗漏“穿过”，恰好改变了本页关于成交确定性的核心区别。
- PDF p2511 中文把卖出限价统一写成“限价空单”，但多头在前高挂出的也是卖出限价止盈，不是新开空仓；应区分 sell-to-close 与 sell-to-open。
- PDF p2512 同理，空头止盈的 buy limit 不是“限价多单”开多；中文按订单方向命名会掩盖其平仓职责。
- PDF p2514 的 `software creates, but does not place` 应译为“创建但暂不发送/激活”，中文两处都写“创建”，丢失状态差异。
- PDF p2515 的 entry sell stop 与 protective buy stop 在中文都被称为“止损”，需要按入场/退出职责重写。
- PDF p2516 英文 `sell order` 是原页方向笔误，中文已按“多单成交”纠正。
- PDF p2518 把 buy market 译为“做多市价单”、sell market 译为“做空市价单”不完整；它们也可分别用于平空和平多。
- PDF p2520 中文“一篮子订单”出现重复断句，实际意思是市价退出后自动取消原 bracket；不是再创建新的 basket。
- PDF p2521 的“下一 K 线开盘后一秒内”只是示例成交时间，不是市价单的保证。
- PDF p2522–2523 为同一页拆分，p2523 补齐了其他订单类型的英文/中文内容。
- PDF p2500–2501、p2505、p2510、p2513、p2517、p2525–2526 为封面、目录、文字拆分页或结束页，没有新增独立论据。

# 订单流模型

## 它解决什么问题

Brooks 用价格行为推断订单压力。交易者不需要知道真实订单簿，也可以通过 K 线、突破、触发、跟进和失败来理解哪一边正在被迫行动。

## 核心思想

价格越过关键位置时，可能触发：

- 顺势交易者的 stop entry。
- 反方交易者的 protective stop。
- 限价交易者的入场。
- 获利了结。
- 被困交易者退出。

触发本身不等于成功。真正重要的是触发后市场是否接受这个方向。

## 读订单压力

常见线索：

- Signal bar 高低点是否被触发。
- Entry bar 是否强。
- 突破后是否有 follow-through。
- 回调是否浅。
- 反方是否快速收回关键位置。
- 哪一边在失败后被迫退出。

订单流不是猜庄家或操纵，而是把价格行为翻译成交易者行为。

## 与其他概念的关系

Order flow 连接 context 和 setup。Context 说明环境是否支持交易，order flow 说明触发后哪一边正在行动，setup 再把它们合成交易计划。

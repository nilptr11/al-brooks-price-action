# Brooks 价格行为执行手册

> **状态：Normative**
>
> 本文件是仓库内入场、无效点、止损、目标、仓位、Trader's Equation、管理和退出的唯一执行权威。它是人的执行流程，不是软件状态协议。交易记录只需要保留原始计划、真实成交和按时间追加的更新。官方来源与核对状态见 `reference/official_sources.md` 和 `reference/audit_matrix.md`。

## 使用方式

执行时从上到下只走一条链：

```text
读市场 -> 选交易命题 -> 检查控制权、位置和空间 -> 等当前触发
-> 写完整入场计划 -> 检查入场数学
-> 允许下单 -> 已挂单 -> 实际成交
-> 观察结果 -> 检查继续持有是否值得 -> 持有 / 减仓 / 退出
```

本文保留 `M/K/C/T/D/Q/P/E/A` 等简称，方便复盘时定位问题；表单和动作优先使用自然语言。所有编号、`D_init/D_live`、`Q_entry/Q_hold`、`G1/G2/G3`、`PT1/PT2`、`MaxLoss`、`PlannedLoss` 和 `OpenRisk` 都是仓库内部记法，不是 Brooks 官方编号。

不可违反的边界：

1. 入场计划和入场数学未完成，不下单。
2. “允许下单”不等于经纪商已经接受；“已挂单”不等于已有仓位；只有实际成交才激活 P1–P4。
3. 部分成交后，已成交数量按持仓管理，未成交余单继续按待成交订单管理。
4. 未成交前若 entry、stop、target、size 或时间条件改变，先撤销旧订单，再重写计划。
5. 实际成交后保留原始入场计划，不用后续信息覆盖；每次成交和管理变化按时间追加。
6. 追加记录不是扩大原合同的许可：不得临场拉远目标、升级管理尺度、补加 runner / TBTL、加入未预设数量或改写交易命题来让持仓数学合格。
7. 减仓或退出决定不等于已经成交；实际数量只能按真实 fill 改变。
8. 原交易命题终止后，先让原仓位归零；新交易重新从候选和入场计划开始，不把旧仓位过户给新计划。

计划尚未成交但必须改变时，在旧计划上记录“撤单”和时间，确认旧订单已无活动数量后另起一份当前计划；不需要版本号。成交后原计划只读，后续全部写入按时间追加的成交与持仓更新。

按时间追加只解决可追溯性，不自动授权修改合同。成交后允许的更新只有：按预写数量执行 PT1 / PT2；在预写条件成立时激活条件式 PT2 / runner；按预写 trailing 方法管理 G3 runner；缩短目标；向保护方向移动 stop；减仓或退出。取消某批次目标时，该批次必须退出或转入原计划已经写明的 runner 管理，不能无目标继续持有。禁止放宽最外侧 stop、拉远固定 PT1 / PT2、补加新目标 / runner / TBTL、把 G1/G2 临场升级为 G3、加入未预设 scale-in，或把 P1–P4 改名后继续持有。新机会必须等原计划实际数量归零后另起计划。

| 层级 | 只回答什么 |
| --- | --- |
| M 市场读取 | 当前基础结构、转换阶段和会实质改变判断的关键证据是什么？ |
| K 候选命题 | 准备验证哪一种交易命题，方向是什么？ |
| C 候选过滤 | 对该方向，控制权、位置和第一目标空间是否值得继续？ |
| T 当前触发 | 现在可以用什么订单或市场行为入场？ |
| D_init 初始计划 | entry、失效、stop、仓位、目标、管理和时间条件是什么？ |
| Q_entry 入场数学 | 现在是否值得承担新风险？ |
| 订单状态 | 只是允许下单、已挂单，还是已经部分或全部成交？ |
| P active plan | 实际成交后正在持有哪一种交易命题？ |
| D_live 实时记录 | 真实仓位、active stop、剩余目标、风险和时间是什么？ |
| E 市场结果 | 触发后出现接受、失败、恢复还是混乱？ |
| Q_hold 持仓数学 | 相比立即退出，是否仍值得持有拟保留数量？ |
| A 动作 | 实际数量保持不变、减少后仍非零，还是归零？ |

## 市场读取与候选

### M：基础状态和叠加阶段

基础状态三选一：

| 状态 | 执行含义 | 默认观察 |
| --- | --- | --- |
| M1 趋势 / 紧通道 | 一方持续控制，回调浅，反方失败快。 | K1；离开重要旧边界时观察 K2。 |
| M2 宽通道 / 弱趋势 | 仍有方向，但回调深、双边交易增加。 | 好位置 K1；通道边缘 K3；结构转换后 K4。 |
| M3 交易区间 | 价格围绕公平区域双向测试。 | 边缘 K3；突破 K2；中部无候选。 |

叠加阶段可以与基础状态并存：

| 阶段 | 最低条件 | 移除或转换 |
| --- | --- | --- |
| M4 突破尝试 | 价格越过有意义的旧边界。 | 接受后转向新的 M1/M2；失败后回到原状态或 M3。 |
| M5 突破模式 | 任一方向突破都可能获得跟进，方向尚未选择。 | 一个方向被接受后移除；普通无空间区间只保留 M3。 |
| M6 高潮行为 / 转换 | 过快过远、overshoot 或控制权可能变化。 | 原趋势恢复、市场区间化，或反向建立新结构后移除。 |

M4 的强收盘和 follow-through 是质量证据，不是 breakout 的最低定义。M6 不是反转信号；confirmed climax 多数先形成 trading range，只有控制权转移证据足够时才观察 K4。

M 只记录会实质改变当前判断的证据，不要求为每一种可见现象建立专项地图。Gap 若改变控制权、位置、突破接受/拒绝、路径障碍或目标，才作为关键结构证据记录其类型、周期、比较边界、当前状态和接受/拒绝证据；无关时不需要填写。Gap 不单独许可交易，也不生成新的候选或计划。

Gap 的开放、进入或回补都只是条件证据，不能单独确认突破接受、失败或反转。Body、micro measuring、moving-average、measuring、exhaustion 与 gap reversal 的定义和训练案例分别见 [`05_patterns/06_gaps.md`](05_patterns/06_gaps.md) 与 [`07_scenarios/02_gap_scenarios.md`](07_scenarios/02_gap_scenarios.md)；其中 measuring 与 exhaustion 的结果名称不能在事前当成已确认事实。

### K：四个候选命题

| 候选 -> 成交后计划 | 正在下注什么 | 第一目标的边界 |
| --- | --- | --- |
| K1 -> P1 趋势延续 | 既有控制方恢复，回调或反方尝试失败。 | 原趋势恢复后的前高前低、前 swing 或第一现实 magnet。 |
| K2 -> P2 突破跟随 | 市场接受离开旧价格区域的尝试。 | 新区域中的第一现实阻力/支撑；有清晰测量结构时可使用投射目标。 |
| K3 -> P3 边缘失败 / 回归 | 边缘测试或突破失败，价格回到旧公平区域。 | 中轴、均线、前 swing 或最近公平区域。 |
| K4 -> P4 控制权反转 | 原控制方失去控制，反方建立可持续 swing。 | 必须有反向 swing 空间，不能只到公平区域。 |

没有清晰候选、多个候选无法排序或位于区间中部时，本次结果是 P0：不激活交易。

P1 与 P2 的边界看交易命题，不看方向是否相同；P3 与 P4 的边界看目标是否只是回归公平区域，还是控制权已经转移并存在反向 swing。

若相关 gap 会实质改变判断，被接受时仍按交易命题区分 K1 与 K2，被拒绝并回到旧公平区域时才支持 K3。Potential exhaustion、gap reversal 或 gap 回补都不能单独生成 K4。

### C：候选过滤

对每个候选方向分别回答：

1. 控制权或失败行为是否支持该命题？
2. 价格是否位于支持该命题的位置？
3. 到第一现实目标是否有足够空间？
4. 大周期价位、旧价格区域、如相关的 gap 边界及其他支撑阻力，是路径障碍还是目标；是否压缩空间？
5. 流动性和时段是否压缩空间？

任一答案不清楚就不继续设计该方向；一个方向不合格不自动否定另一方向。

### T：当前触发

| 触发 | 常见用途 |
| --- | --- |
| T1 顺势恢复 | H1/H2、L1/L2、旗形、反方 failed entry；背景强度不能替代当前恢复触发。基础权限用 stop entry。 |
| T2 突破 | Breakout mode 初始触发、follow-through、突破点回踩守住或 failed failure。 |
| T3 边缘 / 回归 | 边缘或突破失败后收回旧区域，再由 failed entry、二次信号等触发 stop entry；limit probe 需高级权限。 |
| T4 反转 | 成熟趋势中出现结构破坏、测试失败、二次反转或完整 MTR 触发/确认。Climactic behavior 不能替代这些条件。 |

预判型和确认型触发必须分别重写入场价格、止损、目标和数学，不能沿用更好价格下的旧判断。

订单成交前只能称 prospective signal bar；实际 fill 后，setup 的最后一根才取得 signal bar 角色。

## 入场计划

### 执行权限

权限在时段前设定，不由单个 setup 临时产生：

| 权限 | 默认允许 | 默认禁止 |
| --- | --- | --- |
| 基础权限 | Stop entry；G2/G3；P1、P2、确认型 P3、完整 MTR。 | G1 scalp、limit probe、close entry、scale-in、早期 K4。 |
| 高级权限 | 基础项目，以及时段前逐项写明的高级行为。 | 任何未单独授权的高级行为。 |

高级权限至少写明项目、市场/周期、数量或风险上限和有效时段。允许 close entry 不等于允许 scale-in。退出、减仓和 protective stop 不受入场权限限制。

### 管理尺度：G 和 TBTL

| 管理 | 定义 |
| --- | --- |
| G1 scalp | 全部计划数量准备在正常 pullback 前于近端目标退出；仅限高级权限，且保守 reward 不得小于 risk。 |
| G2 small swing | 使用有限的 PT1 和可选 PT2 批次；不要求结构 runner。 |
| G3 swing | 至少一个批次作为 runner，预先允许穿越正常 pullback，由结构 trailing 或有依据的 PT2 管理。 |

TBTL 只能附着在 G2/G3 的一个非零保留批次上。它提供时间和腿数预期，不是价格目标，也不能修复不合格的入场数学。

### 从交易命题到 protective stop

`Invalidation` 是否定交易命题的市场事实；`stop anchor` 是图上代表该事实或最终风险边界的结构价位；`protective-stop trigger` 是发给经纪商的触发价；`最差预计止损成交价` 是风险计算使用的保守 fill。四者相关但不等同。

可选止损方案：

| 方案 | 合格条件 | 常见锚点 |
| --- | --- | --- |
| Signal / pattern stop | prospective signal bar 或入场形态的另一端确实代表当前触发失败，而不是正常测试。 | Signal bar 极值、二次入场形态极值、小型失败结构。 |
| Structure / swing stop | 交易命题允许穿越普通 signal bar，但不能穿越完整回调、突破或测试结构。 | Pullback / flag 极值、突破腿底/顶、failed breakout 极值、major higher low / lower high、旧极值测试。 |
| Wide structural stop | 更宽的 price-action 结构仍支持命题，且计划愿意用更小仓位承受完整测试。 | 较大 swing、区间外侧、完整突破或反转结构外。Wide stop 不自动授权 scale-in。 |

定价顺序：

1. 先写命题失效事实，以及该命题必须容许的正常测试。
2. 列出所有合理 stop anchor，再为当前计划选择其中一个方案和一个初始 protective-stop trigger；该 trigger 从首个成交起生效，也是本计划此后不得越过的最外侧边界。若有两个或三个合理 stop，它们各自形成独立的 entry、size、target、概率和 Q_entry，不能取窄 stop 的仓位、宽 stop 的存活率或另一方案的目标拼成一笔交易。
3. 多头 stop trigger 必须严格低于所选锚点，空头必须严格高于所选锚点。超出锚点的距离使用时段前设定的品种 buffer；未另设 buffer 时至少越过一个最小报价单位。
4. 预计滑点不用于拉远结构锚点。它用于从 stop trigger 推导最差预计止损成交价：多头按更低 fill，空头按更高 fill；风险和仓位使用该预计成交价。
5. 合理 stop 太远时缩小仓位、等待更好触发或不交易；不能用 money stop、固定点数或理想 R 倍数把 stop 塞进正常噪声。
6. 单一价位或容易画出的边界（如 gap 边缘/中点、均线或整数位）只有在越过它确实否定当前命题时才可成为 stop anchor；不能替代完整回调、失败突破或 swing 结构。
7. Protective stop 是灾难边界，不替代提前退出。出现预写的合理反向信号、强反向 breakout 或命题失效时，可以在 stop 触发前退出。
8. 下单前锁定上述初始 stop；首个成交后不得为“多给时间”而放宽。Stop 只能因新结构形成而向保护方向移动，breakeven 不是固定时间动作。

D_init 写有 stop 价格不等于已经受到保护。任何入场或加仓成交后，必须确认覆盖全部实际持仓的 protective-stop 订单已被经纪商接受。Stop 被拒绝、意外撤销、数量不足或与真实仓位不一致时，立即恢复保护；无法恢复时全部退出。

### 最大损失、仓位和加仓

`MaxLoss` 是入场计划允许的最大账户损失，不是成交保证。跳空、停牌、流动性枯竭和异常滑点仍可能造成更大损失；账户和时段风险预算另行设定。

最差可接受成交价是整套计划在该成交价仍同时通过风险、目标、时间和入场数学的边界。下单前按这个价格计算，而不是按理想触发价：

```text
每单位价格风险 = |最差可接受成交价 - 最差预计止损成交价| × point value
每单位未嵌入成本 = 尚未反映在上述两个可执行价格中的预计手续费、点差和滑点
每单位总风险 = 每单位价格风险 + 每单位未嵌入成本
最大允许数量 = 向下取整((MaxLoss - 固定成本) / 每单位总风险)
```

多次入场或 scale-in 必须共用一个从首个成交起即生效的初始 stop，并按其最差预计成交价汇总：

```text
PlannedLoss =
  Σ(每层数量 × (|该层最差可接受成交价 - 共同最差预计止损成交价| × point value
                 + 该层每单位未嵌入成本))
  + 固定成本

PlannedLoss <= MaxLoss
```

执行规则：

- Stop trigger 用于经纪商订单，最差预计止损成交价用于计划数学；已经嵌入预计成交价的 stop slippage 不得再作为成本扣除。
- 未成交数量使用最差可接受成交价；已成交数量改用真实 fill。
- 已发生的手续费和滑点使用实际值，不能与预计成本重复计算。
- 最小交易单位已经超过允许数量时，不交易；不能用过紧 stop 迁就仓位。
- 任何同方向新增数量都视为 scale-in。必须在原计划中写明层级、首个成交起即生效的共同 stop、总数量和 MaxLoss，并具有专项高级权限。
- 每次准备加仓时，新增风险的入场数学、现有仓位的持仓数学和全部数量的总风险必须同时合格；初始入场的判断不能自动授权后续加仓。
- 未预设的新数量只能等原计划结束后，作为新交易重新开始。
- 新增反向数量前，原计划实际仓位必须归零。

部分成交后，已成交数量使用真实 fill，活动余单继续使用其最差可接受成交价。每次入场、加仓或部分退出都立即重算计划损失和“若当前 stop 成交，整笔交易总结果”。

`OpenRisk` 是从当前可执行退出价到 active stop 的最差预计成交价的前瞻暴露，不是整笔交易的总结果。持仓记录必须同时写：

- 当前开放风险；
- 已实现盈亏；
- 若当前 stop 成交，整笔交易总结果；
- 该总结果是否仍不低于 `-MaxLoss`。

如果费用、数量错误或异常成交破坏 MaxLoss，先取消所有未成交增量订单。拟保留数量仍值得持有且减仓后能恢复合规时，可以强制减仓；否则全部退出。减小数量只能修复金额上限，不能把负的每单位期望变成正值。

### 从市场结构到目标价

`目标区` 是市场可能出现获利、反向订单或加速测试的结构区域；`保守可执行退出价` 是计划实际用于订单和数学的价格。Measured move 的数学投射是一个 magnet，不是保证成交的精确点。

目标候选来自：

| 来源 | 常见候选 |
| --- | --- |
| 旧价格与支撑阻力 | 前高前低、日内/前日高低、区间边缘、突破点、重要 swing、大周期价位，以及如相关的 gap 边界与旧价格区域。 |
| 公平区域与通道结构 | 区间中轴、开盘价、均线、趋势线、通道线。 |
| 可计算投射 | Leg 1 = Leg 2、区间/形态高度、breakout height；若计划明确把 gap 用作运动中间参照，也可使用条件式 gap projection。 |
| 动态管理 | 已预写的结构 trailing；它只属于 G3 runner，不替代入场前的有限回报估计。 |

定价顺序：

1. 从最差可接受成交价沿交易方向列出所有重要候选，并按距离排序。弱小、不会实质影响订单行为的价位不必列入；明显支撑阻力、旧极值和目标簇不能省略。
2. 给每个候选标明角色：`路径障碍`、`PT1`、条件式 `PT2`，或仅作 runner 的 trailing 依据。路径障碍影响到达概率，但不自动要求退出。
3. PT1 是计划承诺退出非零数量的第一个现实目标。若 PT1 之前存在重要障碍，该障碍要么成为 PT1，要么必须写明为何当前强度与语境支持穿越；不能静默跳过近端障碍，用远端目标修复 Q_entry。
4. 先写结构目标区，再把保守可执行退出价放在目标区内或交易方向的近侧，并计入最小报价单位、预期流动性和成交概率。Q_entry 使用该保守价格，不使用最理想极值或必须越过目标才可能成交的价格。
5. 多个独立投射与支撑阻力聚集时，把它们视为目标区而不是伪精确单价。各 PT 必须绑定退出数量，不能把同一数量同时计算到多个目标。

### 条件式工具：Measured move 的构造与使用

Measured move 是目标候选的构造方法，不是每笔交易的必经步骤。只有当前计划确实把某个投射用作 PT1、条件式 PT2 或 runner 的有限回报依据时，才调用本节。官方资料给出的是“后续运动约等于此前一腿、区间/形态高度、日内 gap 或 breakout height”等量度关系，并没有规定一套跨所有图表唯一的端点算法。以下公式和冻结端点要求，是本仓库把这些关系转换成可复核计划的执行约定。

每次采用 measured move，都必须在下单前写明类型、测量端点、投射起点和计算值。端点必须来自当时可见的同一结构；若形态边界倾斜或不规则，要记录实际采用的价格和时间，不能只写形态名。以下公式使用带方向的价格差；向上为正，向下为负。

```text
Leg 1 = Leg 2：
  第一腿 A -> B，第二腿起点 C
  Δ = B - A
  投射目标 T = C + Δ

区间 / 形态高度：
  H = 上边界 - 下边界
  向上突破目标 = 上边界 + H
  向下突破目标 = 下边界 - H

Breakout height（复制当前计划选定的完整突破段）：
  Δ_BO = breakout 结束价 - breakout 起始价
  投射目标 T = breakout 结束价 + Δ_BO
  起始价和结束价必须是入场前已标出的可见价格，不能事后在 close、high/low 或不同 K 线之间切换。

Gap-as-halfway projection（仅在入场前已明确把 gap 用作运动中间参照时）：
  运动起点 O，预先选定的 gap 参考价 G（通常为 gap 中点；特定官方案例也可使用靠近运动起点一侧的边缘）
  Δ_gap = G - O
  投射目标 T = G + Δ_gap
```

使用边界：

- Measured move 只提供目标候选，不提供入场许可或到达概率；不用 measured move 的计划只要第一现实目标和数学完整，同样可以合格。概率仍来自 context、突破强度、follow-through、路径障碍和剩余时间。
- 只有把 gap 明确当作运动中间参考时才使用上述 gap-as-halfway 公式；许多 measuring gap 的目标仍主要来自此前区间/形态高度。Gap 必须仍开放且延续证据支持 measuring 而非 exhaustion。`Measuring gap` 只有事后才能确定；入场前只能记录“候选 measuring gap”和条件式投射。
- 强且被接受的突破中，measured move 可以是 PT1；弱突破、通道或交易区间语境中，它常是 PT2，或只是与更近 magnet 汇合的目标区。
- P3 的 PT1 是回到旧公平区域；另一侧边缘只能是条件式 PT2。Measured move 不能把回归命题伪装成 P4。
- P4 必须保留反向 swing 的现实空间；若全部计划只到公平区域，应归 P3。
- G3 runner 可以没有固定最终限价单，但入场数学必须使用保守、有限的回报估计。
- TBTL 是时间/腿数预期，不是价格目标，也不能替代 measured move 的端点。
- 剩余时间不足以到达目标时，不入场；成交后时间耗尽时重新检查持仓数学。
- 成交后固定 PT1 / PT2 只能保留或缩短；取消目标时，对应批次必须退出或转入原计划已经写明的 runner 管理。只有入场计划已经写明的条件式 PT2 / runner 才能在条件成立时启用，G3 runner 只按预写 trailing 方法延续。不得拉远目标或升级管理尺度来让 Q_hold 重新变正。

## 入场数学、订单和持仓数学

### Q_entry：现在是否值得承担新风险

最简两结果近似：

```text
入场期望 = p × 毛价格盈利 - (1 - p) × 毛价格亏损 - 预期全路径成本 > 0
```

- 概率、盈利、亏损、成本、数量、目标和时间必须来自同一份当前计划。
- 毛价格盈利从最差可接受成交价计算到对应批次的保守可执行退出价；毛价格亏损从最差可接受成交价计算到最差预计止损成交价。两者均不包含另列成本；预计更好的 fill 或更理想目标只能作为敏感性情景。
- 预期全路径成本只包含尚未嵌入可执行价格的手续费、点差和滑点，并且只扣一次。若 entry、stop 或 target 已使用含某项成本的可执行价格，该项不得再次列入成本。
- `PlannedLoss` 用于 MaxLoss 约束，已经包含 stop 路径的毛价格亏损和相应成本；它不等于上式单独的“毛价格亏损”。在 stop 情景中，两者应满足 `毛价格亏损 + 该路径未嵌入成本 = PlannedLoss`。
- 多目标、scratch、部分退出和提前退出明显影响结果时，使用互斥且穷尽的多结果期望，概率合计为 1。每个结果要么分别列毛账户盈亏和该路径成本，要么直接列已扣成本的净结果；使用净结果时不得再统一减一次成本。
- 同一仓位到达 PT1 和 PT2 不能作为两个独立、全仓成功结果重复计算。
- 没有可靠校准时使用保守概率区间，不写伪精确小数。
- 只有依赖完美 discretionary exit 才成立，或对小幅假设变化极其敏感时，不下单。

### 订单与实际成交

| 状态 | 人的执行含义 |
| --- | --- |
| 允许下单 | 当前计划和入场数学合格，但尚未成为经纪商活动订单。 |
| 已挂单 | 经纪商已经接受，仍有活动数量；零成交不代表已有仓位。 |
| 部分成交 | 已成交数量成为实际持仓，未成交余单仍是活动订单。 |
| 全部成交 | 该订单计划数量已经实际成交，没有活动余量。 |
| 撤销 / 到期 / 拒绝 | 该订单没有活动余量；此前若有部分成交，实际仓位仍然存在。 |

订单按用途写自然名称：初始单、加仓 1、加仓 2、PT1 减仓、保护性止损、全退出。没有必要使用订单编号体系。

待成交期间：

1. 逐根复核 M、K、C、T、最差可接受成交价、剩余时间和入场数学。
2. 条件不再满足时立即撤单。
3. 需要改变计划时，先撤销旧活动订单，再重写计划；不能让旧订单继续活动。
4. 零成交订单最终撤销、到期或拒绝时，本次结果为 P0/A0。
5. 实际 fill 后立即记录真实价格、数量、成本和 protective stop。
6. 实际 fill 越过最差可接受成交价属于执行异常：立即取消所有未成交的入场和加仓余单，并确认实际持仓已被 protective stop 覆盖；随后用真实 fill 重算风险和 Q_hold。原命题、目标/时间或 Q_hold 任一不合格时 A3；若只有金额超过 MaxLoss，且拟保留数量的 Q_hold 通过，允许强制 A2 至恢复合规，否则 A3；若风险合规且 Q_hold 通过，可以 A1，但复盘仍记为越界成交，不能倒推为合格入场。

部分成交时同时做两件事：

- 对已成交数量，建立持仓记录并检查继续持有是否值得；
- 对未成交余单，继续检查原入场计划、入场数学和总风险。

### Q_hold：相比立即退出，是否仍值得持有

成交后不再用原 entry、已支付成本或已实现盈亏证明继续持有。最简近似：

```text
持仓期望 =
  剩余成功概率 × 从当前可执行退出价到剩余目标的回报
  - 剩余失败概率 × 当前开放风险
  - 继续持有相对立即退出的增量成本
  > 0
```

- 概率根据当前 M、E、剩余目标和剩余时间重新判断。
- 回报使用当前可执行退出价到剩余目标的保守可执行退出价；开放风险使用当前可执行退出价到 active stop 的最差预计成交价。两者和增量成本必须对应同一拟保留数量。
- 现在退出也会发生的共同成本，不能只扣在持有方案上。
- 已平仓数量和沉没成本不进入前瞻比较，但整笔交易仍必须满足 MaxLoss 约束。
- 不同批次分别检查；拟保留数量不能独立通过时，全部退出。
- 同质仓位按比例减小会同比缩小 reward 和 risk，不能改变每单位期望的正负。

## 市场结果与动作

### E：触发后的市场结果

| 事件 | 可观察含义 | 常见作用 |
| --- | --- | --- |
| E1 顺势恢复并跟进 | 原方向触发后有连续性，回调方再次失败。 | 支持 P1；也可能生成新的 K2。 |
| E2 突破被接受 | 强收盘、follow-through、回踩守住突破点或旧边界，或反方无法收回。 | 支持 K2。 |
| E3 突破 / 边缘测试失败 | 触发后缺乏跟进，价格回到旧区域，并获得反向跟进。 | 支持 K3，或恢复 M3。 |
| E4 Failed failure | 先交易 failure 的一方也失败，原突破方向恢复。 | 支持新的 K2。 |
| E5 控制权转移 | 结构破坏、原方向测试失败和反向跟进形成组合。 | 支持 K4。 |
| E6 无跟进 / 混乱 | 双方都没有形成可交易优势。 | 空仓时等待；持仓时重算 Q_hold。 |

E 与交易者是否成交无关。空仓时它更新候选，待成交时它决定保留或撤单，持仓时它改变 Q_hold 和动作。

若入场时使用了 gap 作为关键证据，其开放、进入或回补仍不能单独确认 E2/E3；必须结合旧区域是否被重新接受、同/反方向 follow-through、回踩质量和原交易命题判断。

### A：实际持仓动作

| 动作 | 数量含义 |
| --- | --- |
| A0 wait / no-entry | P0 后无活动订单且持仓为零，继续等待。 |
| A1 hold | 实际持仓数量保持不变；合法推进 active stop 仍属于 A1。 |
| A2 reduce | 目标是数量减少且完成后仍大于零；拟保留数量必须通过 Q_hold。 |
| A3 exit | 目标是该计划实际数量归零。 |

A2 通常来自预设部分目标或风险收缩；异常成交破坏 MaxLoss 时可以强制减仓。推进 stop 本身不是 A2。A2/A3 的退出订单尚未成交时，继续按真实未平数量维护 D_live 和 protective stop；A3 只有在实际仓位归零后才完成。

### 唯一总流程

```text
无清晰候选 / 位置或空间不合格 / 无触发 / 计划不完整 / 数学不合格
  -> P0 / A0

计划与数学合格
  -> 允许下单
  -> 经纪商接受：已挂单
  -> 零成交终止：P0 / A0
  -> 部分成交：已成交数量进入 P；余单继续复核
  -> 全部成交：按真实数量进入 P

持仓
  -> 观察 E
  -> 命题失效：A3，直到真实数量归零
  -> 命题仍成立：检查 Q_hold
       -> 全部持有合格：A1
       -> 预设减仓后保留部分合格：A2
       -> 无合格保留数量：A3
```

### Active plan 更新矩阵

| Active plan | 有利结果 | 失效或退出 | 归零后的新候选 |
| --- | --- | --- | --- |
| P1 | E1 且 Q_hold 通过：A1；预设部分目标/风险收缩点：A2。 | 原趋势恢复命题失效或无合格保留数量：A3。 | E2 后 K2；E5 后 K4；否则等待。 |
| P2 | E2/E4、回踩守住且 Q_hold 通过：A1；预设部分目标：A2。 | E3、快速回旧区域或无合格保留数量：A3。 | E3 后 K3；小回调趋势后 K1；控制权转移后 K4。 |
| P3 | 向公平区域推进且 Q_hold 通过：A1；按原始计划在 PT1 部分退出，且预写 runner 的 Q_hold 通过：A2。 | 到达 PT1 时未预写 runner、突破重新被接受或命题失效：A3。 | E2 后 K2；控制权转移后 K4；否则等待。 |
| P4 | E5、反向跟进且 Q_hold 通过：A1；预设部分目标：A2。 | 原趋势强恢复、反向压力不足或无合格保留数量：A3。 | 原方向恢复后 K1；反向突破接受后 K2；否则等待。 |

## 四个交易计划

以下只写各计划的特有差异；权限、stop、风险、数学和订单边界统一使用前文规则。

### P1 趋势延续

- **命题**：既有方向仍控制市场，当前回调或反方尝试将失败。
- **常见读取**：M1，或 M2 中方向仍在、回调没有破坏结构且位置良好。
- **触发**：T1；基础权限用 stop entry，高级权限才可强趋势 close entry。
- **失效**：回调不再只是暂停，原控制方恢复命题被结构破坏或强反向跟进否定。
- **Stop anchor**：回调、旗形或定义趋势恢复的 swing 外；prospective signal bar 另一端只有在它同时代表该结构失效时才可使用。
- **目标**：PT1 先看前高前低、前 swing 或第一个现实 magnet；持续恢复时，PT2 / runner 可使用 Leg 1 = Leg 2、spike / flag height 或更远结构目标。强趋势多用 G3，弱通道多用 G2。

### P2 突破跟随

- **命题**：市场正在离开旧价格区域，这次突破将被或已经被接受。
- **常见读取**：M4、M5 初始突破、follow-through、回踩守住或 E4。
- **触发**：初始 stop entry，或确认后的再次触发；强 breakout close 需要高级权限。
- **失效**：缺乏跟进、快速回旧区域，或回踩明确失守并出现反向跟进。
- **Stop anchor**：直接追随强突破时放在完整 breakout leg 的底/顶外；回踩确认时放在 retest 极值和突破失败边界外。确认越晚，stop 可能越远，必须同步缩小仓位并重算 Q_entry。
- **目标**：先列出下一明显支撑阻力和其他现实 magnet；若存在清晰测量结构，再计算区间、形态或 breakout height 投射。强且被接受的突破可把投射作为 PT1；较弱突破先在近端 magnet 兑现，投射只作为条件式 PT2。持续接受时可用 G3。

### P3 边缘失败 / 回归

- **命题**：边缘测试或突破失败，价格将回到旧公平区域。
- **常见读取**：M3 边缘、M2 宽通道边缘、M4+E3 或 M5 首次突破失败。
- **触发**：基础权限等待收回旧区域或 failed entry 后用 stop entry；limit probe 需要高级权限。
- **失效**：价格离开旧公平区域并被市场接受。
- **Stop anchor**：失败突破极值、区间边缘或最后一个仍能证明“回到公平区域”命题的 swing 外。Wide-stop 方案必须从开始使用更宽结构并相应缩仓，不能成交后放宽。
- **目标**：PT1 是中轴、均线、开盘价、前 swing 或最近公平区域中第一个现实兑现区；另一侧边缘只能预写为条件式 PT2。基础权限通常 G2。
- **Scale-in**：仅限成熟、足够宽的区间，必须专项授权并从开始计入所有层级和 MaxLoss。

### P4 控制权反转

- **命题**：原趋势方失去控制，反方将建立可持续的反向 swing。
- **常见读取**：成熟趋势、结构破坏、原趋势测试失败和反向持续压力。
- **触发**：基础权限等待完整 MTR；早期 K4 需要高级权限和足够 swing 回报。
- **失效**：原趋势恢复并重新建立控制。
- **Stop anchor**：原趋势旧极值测试、失败回探极值或完整反转结构外。早期 K4 通常需要更宽的结构 stop；强反向 breakout 后的确认入场概率更高，但 entry、stop 和剩余 reward 必须全部重算。
- **目标**：可在均线、前 swing、区间中轴或其他近端障碍部分兑现；若这些只代表回到公平区域，完整计划还必须保留反向 swing 的 PT2 / runner。若 PT1 本身已是可量化的反向 swing，PT2 可以留空；有清晰测量结构时，可使用反向 Leg 1 = Leg 2 或反转结构高度。G2/G3 可附 TBTL。
- 若全部目标只到均线、区间中轴或旧公平区域，应归 P3。

### P0 不激活交易

无清晰 K、C 不合格、无当前 T、入场计划不完整、Q_entry 不通过，或合格订单最终零成交终止，结果都是 P0/A0。P0 不是第五种交易计划，也不是执行错误的同义词；设计合格但按规则未成交仍是正确执行。

## 逐根检查

### 空仓或尚未下单

1. M 及会实质影响判断的关键结构证据是否改变，是否添加或移除 M4–M6？
2. 当前最清晰的 K 和方向是什么？
3. C、T、完整入场计划和 Q_entry 是否同时合格？
4. 不合格时，正在等待什么具体新信息？

### 已挂单或部分成交

1. 订单状态、计划数量、活动数量和真实成交数量是否准确？
2. 原候选、最差可接受成交价、剩余时间和 Q_entry 是否仍成立？
3. 条件变化时是否先撤单，再重写计划？
4. 部分成交时，已成交数量的 Q_hold 与余单的 Q_entry 是否分别通过？

### 持仓

1. 出现了哪个 E，原交易命题仍成立、削弱还是失效？
2. 实际数量、已实现盈亏、active stop 触发价与最差预计成交价、剩余目标的保守退出价/时间和风险是否准确？
3. 相比立即退出，拟保留数量是否仍值得持有？
4. 当前应 A1、A2 还是 A3；退出订单是否已经真实成交？
5. 如准备 scale-in，新增风险、现有持仓和总 MaxLoss 是否同时合格且已有专项权限？
6. 原计划失效时，是否先让实际数量归零，再考虑新候选？

# Al Brooks 价格行为执行手册

> **状态：Normative**
>
> 本文件是仓库内唯一的交易执行权威。前半部是学习者和基础执行者使用的核心流程；后半部高级附录只在真实订单、部分成交、复杂退出或预设加仓确实存在时使用。Al Brooks 概念与仓库内部保守政策必须分开理解。

## 1. 使用边界

本手册把 Al Brooks 的读图理念转换成一份可复核的人类交易计划，不是软件状态机，也不保证任何交易盈利。

真正的 Al Brooks 主线很短：先判断 market cycle，再结合 context、位置和买卖压力理解 setup；随后用合理的 protective stop、现实目标和 Trader's Equation 决定是否值得承担风险；成交后继续观察 price action，原交易 premise 不再成立时及时退出。

以下属于仓库的保守执行政策，不是 Al Brooks 官方编号或术语：

- 下单前书面固定 entry、失效事实、stop、目标、仓位和管理方案。
- 不因持仓亏损而放宽 stop、增加未计划数量或拉远目标。
- 原始计划与真实成交分开保存，不能用后续走势覆盖当时判断。
- 复杂入场和加仓只有在事前授权、单独计算并能承受完整风险时才允许。

本仓库不替用户设定具体账户数值。真实交易前必须另行确定账户、单笔和时段风险上限、停止交易条件及相关敞口限制；未设定这些上位风险边界时，本手册只能用于模拟和复盘。

## 2. 核心五问

每个候选只回答五个问题：

1. **市场与位置**：当前首先是 trend 还是 trading range；若是 trend，更像 breakout、tight channel 还是 broad channel？价格位于什么支撑阻力、区间位置或旧极值附近？
2. **交易想法**：准备验证的是趋势恢复、突破被接受、边缘失败后回到公平区域，还是控制权反转？哪些可观察证据支持，哪些证据会否定？
3. **入场与失效**：什么具体价格行为构成 setup 和入场条件？在哪里可以确认 premise 已错？Protective stop 应放在哪里？
4. **数学与仓位**：第一现实目标在哪里？概率、风险、回报、成本和仓位是否属于同一方案，并且在保守假设下仍合理？
5. **成交后管理**：目标、premise 失效、反向强突破、trailing 和时间退出分别如何处理？没有新证据时准备承受什么正常回调？

任何一问说不清，结果都是等待或不交易，不需要再分配内部代码。

## 3. 学习者默认范围

学习和建立一致性时，默认采用较窄的能力范围：

- 主要寻找顺势或清晰突破后的 stop-entry swing。
- 使用足以代表 premise 失效的 price-action stop，并按完整 stop 距离缩小仓位。
- 优先选择至少有约两倍风险潜在回报的 swing；高概率强突破可接受较小相对回报，但必须重新按较差 entry 和较远 stop 评估。
- 使用足够小、不会因担心亏损而破坏管理的仓位。
- 回避区间中部、紧密交易区间、普通逆势 scalp 和没有现实目标空间的信号。
- Limit entry、close entry、scale-in、早期逆势反转和高频 scalp 视为高级行为，不属于默认学习路线。

这不是说高级方法无效，而是把能力范围与学习阶段匹配。官方依据登记为 `SRC-MANUAL`、`SRC-STOP-ORDERS`、`SRC-POSITION-SIZE` 和 `SRC-RISK-113`。

## 4. 一份核心交易计划

每个准备下单的候选只保留一份当前计划。可以写在纸面、表格或日志软件中，但至少包含以下内容：

```text
[基本信息]
日期、时间、品种、方向、周期、时段：
账户/单笔/时段风险边界及本次允许损失：
价格单位、最小变动、手续费与保守滑点假设：

[市场与交易想法]
市场状态、控制方向和关键位置：
交易想法（趋势恢复 / 突破接受 / 回归公平区域 / 控制权反转）：
支持证据、对手方向仍然成立的理由、尚待验证的信息：

[入场与风险]
Setup 与具体入场条件、订单方式、最差可接受成交价：
Premise 失效事实与必须容许的正常测试：
Protective-stop 结构锚点、订单触发价、保守预计成交价：
仓位和本次最坏计划损失：

[目标、数学与管理]
路径上的重要障碍：
第一现实目标、保守退出价和退出数量：
可选 runner / 更远目标及其启用条件和数量：
概率区间、保守盈利、保守亏损、成本与结论：
目标、premise 失效、反向强突破、trailing、时间退出的条件与动作：

[订单与真实成交]
提交、接受、部分成交、全部成交、撤销或拒绝的时间与数量：
真实成交价、费用、实际持仓和 protective stop 覆盖情况：

[管理与退出]
新出现的价格行为、原 premise 是否仍成立、采取的动作：
减仓或退出订单的真实成交、剩余数量、active stop 和剩余目标：
```

未成交前，条件改变时先取消旧活动订单，再重写计划。成交后保留原计划，只追加真实成交、新出现的价格行为和实际动作。

## 5. 市场状态与四种交易想法

### 市场状态

Al Brooks 的基础分叉只有 trend 与 trading range。Trend 再观察它处于 breakout、tight channel 还是 broad channel；这些状态构成从单边到双边交易逐步增加的连续谱。

Breakout mode、climactic behavior、gap、day type 和大周期价位都只是可能改变当前判断的附加证据，不是每次必须填写的第二套状态分类。

### 四种交易想法

| 交易想法 | 正在验证什么 | 常见失效事实 | 第一现实目标 |
| --- | --- | --- | --- |
| 趋势恢复 | 既有控制方会在回调或反方失败后恢复 | 回调演变为结构破坏或出现强反向跟进 | 前高前低、前 swing 或第一现实 magnet |
| 突破接受 | 市场会离开旧价格区域并接受新价格 | 突破缺乏跟进并明确回到旧区域 | 新区域中的第一支撑阻力；结构清晰时可考虑 measured move |
| 回归公平区域 | 区间边缘或突破尝试失败，价格回到旧公平区域 | 市场离开旧区域并获得持续接受 | 中轴、均线、开盘价、前 swing 或最近公平区域 |
| 控制权反转 | 原趋势失去控制，反方建立可持续 swing | 原趋势恢复并重新建立控制 | 近端障碍可部分兑现，但计划必须存在现实反向 swing 空间 |

同一图表可能同时支持多种解释。只选择一份能够明确写出失效、stop 和目标的交易想法；无法排序时等待。

## 6. Setup、入场和可观察证据

Setup 是 context 与一根或多根 K 线组成的入场依据，不是完整交易计划。形态名称本身不许可交易。

下单前重点观察：

- 价格是否位于支持该想法的位置，而不是区间中部或已接近目标。
- 突破或恢复是否有足够强度和 follow-through。
- 回调是否符合当前市场状态，还是已经改变了状态。
- 对手方向是否仍有合理的 context、位置和目标。
- 触发后若出现 weak/disappointing entry，需要什么进一步证据才确认 failure；不要把普通 pullback 立即叙述成 trapped traders。

可以用“哪些交易者可能被迫行动”帮助理解 failure，但无法从图表知道真实对手方身份或具体订单来源。执行判断必须落回可观察的价格、跟进、失败和位置。

## 7. Protective stop、目标和仓位

四个价格概念需要分开：

1. **Premise 失效事实**：什么市场行为说明交易想法不再成立。
2. **结构锚点**：图上代表该失效或灾难边界的 swing、signal bar、突破腿或区间边界。
3. **Protective-stop 触发价**：实际发给经纪商的 stop 价格。
4. **保守预计止损成交价**：计入滑点后用于计算仓位和最坏结果的价格。

同一图表可能存在多个合理 stop。每个 stop 都会改变仓位、概率、目标空间和管理，因此必须作为不同方案评估，不能拼接窄 stop 的仓位和宽 stop 的存活率。

合理 stop 太远时，缩小仓位、等待更好位置或放弃。不得为了得到漂亮的 R 倍数，把 stop 放进正常波动；也不得成交后为了挽救交易而放宽。

目标从当前位置沿交易方向按距离检查：

- 先列最近的重要支撑阻力和路径障碍。
- 第一现实目标必须真实退出非零数量，并进入初始 Trader's Equation。
- Measured move 只是目标候选，不保证到达；具体构造见[支撑阻力与目标](learning/02_context/01_support_resistance_targets.md)。
- 更远目标或 runner 只有在当前结构和剩余时间支持时才计入计划。

仓位服从 stop 距离和预设损失边界，而不是服从信心。计算至少使用最差可接受 entry、保守止损 fill、数量和未嵌入价格的成本。

## 8. Trader's Equation

最简近似为：

```text
期望 = 成功概率 × 保守盈利 - 失败概率 × 保守亏损 - 预期成本
```

概率、entry、stop、target、数量、管理方式和时间必须来自同一方案。没有可靠统计校准时使用宽而保守的概率判断，不写伪精确小数。

Al Brooks 的通用启发是：不确定时把胜率放在约 40–50% 的范围，并寻找至少约两倍风险的潜在回报；强突破等较高概率环境可以接受约一倍风险的目标，但通常要付出更差 entry、更远 stop 或更少剩余空间。

多目标、scratch 和提前退出明显影响结果时，可以做多结果估算，但学习阶段不要求构造看似精确的完整概率树。只要结果依赖无法校准的细小概率差、完美 discretionary exit 或理想成交，就视为不合格。

## 9. 下单、成交与保护

必须区分：

| 状态 | 含义 |
| --- | --- |
| 计划合格 | 可以提交订单，但经纪商尚未接受 |
| 活动订单 | 经纪商已接受，仍有未成交数量 |
| 部分成交 | 已成交数量是实际持仓，余量仍是活动订单 |
| 全部成交 | 计划数量已成交，没有活动入场余量 |
| 撤销、到期或拒绝 | 没有活动余量；此前部分成交形成的持仓仍然存在 |

任何入场成交后，立即确认全部真实持仓已被经纪商接受的 protective-stop 订单覆盖。Stop 被拒绝、意外撤销、数量不足或无法恢复时，退出未受保护的持仓。

退出决定不等于已经退出；只有真实 fill 才改变实际数量。订单未成交前，仍按真实持仓维持保护。

## 10. 成交后的管理

计划用于防止临场扩大风险，不用于阻止交易者接受新信息。成交后继续读取价格行为，并区分三种情况：

| 新信息 | 动作原则 |
| --- | --- |
| 原 premise 仍成立，只是正常回调或短暂失望 | 按入场前确定的 scalp/swing 方式管理，不因焦虑随意改变尺度 |
| 出现原计划已有的目标、trailing 或时间条件 | 按计划数量执行，并以真实成交更新持仓 |
| 强反向突破、持续反向跟进或其他可观察事实使 premise 不再成立 | 及时减仓或退出；不因该情形没有逐字预写而继续依赖希望 |

任何情况下都不得：

- 向风险方向移动 protective stop。
- 拉远固定目标来修复亏损。
- 临时增加未计划仓位或把 scalp 改名为 swing。
- 用新的形态名称把已经失效的持仓包装成另一笔交易。

若原 premise 已失效，先让实际仓位归零；新的反向机会另建计划。若原 premise 仍成立且没有可执行的新信息，继续按原管理方式持有。

## 11. 最小事件检查

### 尚未成交

1. 当前市场状态、位置和交易想法是否仍成立？
2. Entry、失效、stop、第一目标、仓位和数学是否仍属于同一方案？
3. 条件改变时是否已经撤销旧活动订单？

### 已经持仓

1. 真实数量是否受到完整 protective-stop 保护？
2. 新价格行为是正常回调、计划事件，还是已经使 premise 失效？
3. 采取的动作是否降低或维持原风险，而没有扩大原交易？
4. 退出订单是否真实成交，剩余数量和 stop 是否准确？

复盘入口见[训练与复盘](training/README.md)，陌生词查[核心术语表](reference/glossary.md)。官方来源和核对状态属于维护证据，统一从[参考与来源](reference/README.md)进入。

## 高级执行附录

本节不属于默认学习路线。只有相关行为或订单情况实际存在时才使用。

### 主动高级行为

#### 多目标与 runner

每个目标必须绑定退出数量。同一数量不能同时按多个目标计算盈利。Runner 可以使用结构 trailing 而没有固定最终限价单，但初始数学仍需采用有限、保守的回报估计。

取消某批次目标时，该批次必须退出或转入原计划已经允许的 runner 管理，不能无目标持有。TBTL 只能作为时间和腿数预期，不能替代价格目标。

#### 预设加仓

加仓必须同时满足：

- 原计划在首个订单前已经写明每层触发、价格和数量。
- 所有层从第一笔成交开始共用明确的最外侧风险边界。
- 每层新增风险和全部层最坏总结果均在账户及单笔限额内。
- 加仓是计划的一部分，不是亏损后的临场补救。

未同时满足时禁止增加数量。

### 真实订单异常

#### 部分成交

部分成交后同时处理两件事：

- 已成交数量立即成为真实持仓并获得完整 protective-stop 覆盖。
- 未成交余单继续按待成交订单管理；原 setup、价格边界、剩余时间或总风险改变时撤销余单。

风险计算使用真实 fill 加活动余单的最差允许价格，不能用计划均价替代真实库存。

#### 异常成交与保护失败

若成交越过最差可接受价格、数量错误或风险超限：

1. 取消所有未成交的新增风险订单。
2. 确认真实持仓已有完整 protective-stop 保护。
3. 用真实 fill 和实际数量检查最坏结果。
4. 原 premise、目标空间或上位风险边界不再合格时减仓或退出。
5. 复盘记录执行异常，不用最终盈亏反向证明该成交合理。

#### 最低订单日志

复杂订单至少按时间追加：用途、状态、计划数量、活动数量、本次和累计成交数量、成交价、费用、实际持仓、active stop，以及若 stop 成交的保守总结果。

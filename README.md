# Brooks 价格行为学文档

本目录用于整理 Brooks 价格行为学的核心阅读框架。它不是交易系统、指标参数、回测报告或信号清单，而是一套用来读市场、构造交易想法、管理风险的概念地图。

## 文档边界

本文档只整理 Brooks 价格行为阅读的核心思想和训练流程：

- 关注 market cycle、context、订单触发、失败、被困交易者、概率、风险和管理。
- 不承诺覆盖 Brooks 课程、书籍和交易室里的所有细节。
- 不提供具体市场、品种、周期或仓位建议。
- 不把任何形态写成自动交易信号。

如果某个概念在多处出现，只有 `concept_index.md` 指向的文件是主定义。其他文件只做应用、对比或训练说明。

## 收敛入口

`synthesis/00_core_framework.md` 是本仓库的压缩入口。先用它把所有术语放回同一条阅读链条，再按 `concept_index.md` 回到主定义文件。

- 快速复盘或实盘前：先看 `synthesis/00_core_framework.md`、`synthesis/03_strategy_branches.md` 和 `synthesis/02_review_questions.md`。
- 系统学习：按下面编号路线阅读。
- 扩展概念：先查 `concept_index.md`，不要为同一概念新增第二个主定义。

## 阅读路线

1. `00_method/`：先理解 Brooks 方法为什么不是形态识别。
2. `01_market_cycle/`：判断市场处于趋势、通道、交易区间、突破、突破模式还是高潮转换。
3. `02_context/`：理解 Always In、控制权、位置、目标、多周期和时段。
4. `03_order_flow/`：把 K 线行为翻译成订单触发、跟进、失败和被困交易者。
5. `04_setups/`：把背景、触发、失效点、目标和管理合成完整交易想法。
6. `05_patterns/`：把 H/L 计数、楔形、双顶双底、最终旗形、三角形和缺口当成语言，而不是信号。
7. `06_trade_management/`：处理入场、止损、目标、scalp/swing、TBTL、加减仓和心理纪律。
8. `07_scenarios/`：用开盘、缺口、交易日类型和逐根模板训练。

## 分析结论目录

`synthesis/` 用于整理复盘、压缩和归并后形成的分析结论。它不替代主定义文件，而是把分散概念收敛成更少的判断问题。

## 核心检查问题

任何形态、setup 或入场点，都必须回到这些问题：

1. 当前 market cycle 是什么？
2. 多空哪一边有控制权，优势是否足够明显？
3. 价格在什么位置，附近有什么目标或支撑阻力？
4. 这次触发会让哪一边入场或退出？
5. 触发后有没有跟进？
6. 如果失败，谁会被困？
7. 止损、目标、概率、风险回报和成本是否合算？
8. 这笔交易应该按 scalp、swing、TBTL 还是放弃来管理？

如果这些问题答不清楚，形态名称再正确，也不是完整的 Brooks 阅读。

# 复盘检查清单

> **状态：Training / Derived**
>
> 本清单从当前[执行手册](../README.md#权威层级)派生，用于训练和复盘，不新增执行规则；若字段或边界不一致，以执行手册为准。

## 使用方式

每次只复盘一段走势或一笔交易。先记录当时可见信息，再看后续结果；不要用事后价格替当时的 K、概率或目标辩护。某一层答不清楚时直接记“不清楚”，不要用形态名称补洞。

## 空仓：M、K、C

1. 基础状态是 M1 趋势/紧通道、M2 通道/弱趋势，还是 M3 交易区间？
2. 当时是否存在 M4 突破尝试、M5 突破模式或 M6 高潮/转换叠加？证据是什么？
3. 当前最清晰的候选是 K1 趋势恢复、K2 突破接受、K3 回归公平区域，还是 K4 控制权转移？方向是什么？
4. 对该候选，控制权、位置、第一目标空间和时段是否形成 C1？
5. 是否在区间中部、barbwire、紧区间或多个候选无法排序，因此应该 P0？

## 订单批准前：T 和 D_init

1. 本时段是基础权限还是逐项授权的高级权限？当前 stop、limit 或 close entry 是否在权限内？
2. 当前真实可执行的 T 是什么；`Entry`、预计成交区和最不利可接受成交边界/滑点是什么？
3. `Invalidation`：哪个可观察事实会否定当前 K？
4. `Protective stop`：订单放在哪里，为什么在 invalidation 外？Signal-bar stop 是否真的代表结构失效？
5. `Risk & size`：`MaxLoss`、适用的 OpenRisk 上限及处置、每单位风险、仓位、成本/滑点和所有预设加仓后的总风险是多少？
6. `PT1`：第一个明显、现实的目标区域在哪里？附近障碍是否已经压缩空间？
7. `PT2 / runner`：是否有独立证据，还是为了让交易看起来合格而选择的远端目标？
8. `G`：G1、G2 还是 G3？部分退出比例和 trailing 方法是什么？TBTL 是否只是附加的时间/腿数预期？
9. `Early exit / time`：哪些 E、结构或时间条件会触发 A2 / A3？

任一必填项不清楚都表示 D_init 不完整，本次候选记 P0 / A0，而不是“入场后再决定”。

## Q_entry 和待成交订单

1. 概率是否对应同一套 entry、stop、PT1/PT2 和 G，而不是理想形态的胜率？
2. Reward、loss 和 costs 是否使用同一单位？
3. 部分止盈是否按比例计算，runner 是否使用有限的保守回报？
4. 合理止损太远时是否缩小仓位或放弃，而不是把 stop 塞进正常噪声？
5. 对概率、滑点或目标做小幅保守调整后，EV_entry 是否仍大于零？若否，Q_entry 不通过，应记 P0 / A0。
6. Q_entry 通过后，是否只把订单记为“已批准 / 待成交”，而没有提前激活 P？
7. 待成交期间是否逐根复核 M、K、C、T、最不利可接受成交边界、剩余时间和 Q_entry，并在失效时立即撤单？
8. 部分成交后，是否让已成交部分进入 D_live / Q_hold，同时让未成交余单继续接受 D_init / Q_entry 和总风险预算约束？

## 持仓：D_live、E、Q_hold 和 A

1. D_live 是否记录真实成交均价、剩余数量、active stop、剩余目标、剩余时间、OpenRisk 和当前管理状态？
2. 触发后出现 E1 顺势恢复、E2 突破接受、E3 失败、E4 failed failure、E5 控制权转移，还是 E6 混乱？该 E 即使空仓也应能被同样观察到吗？
3. 原 active plan 的命题仍成立、只是削弱，还是已经失效？
4. 当前 `p_remaining`、从可执行退出基准到剩余目标的 reward、OpenRisk 和 `Cost_hold_delta` 是否使用同一拟保留仓位与同一单位，并避免重复计算价差/滑点？
5. 当前动作应为 A1 hold、A2 reduce 还是 A3 exit？A2 是否来自 D_init 预案，且拟保留批次自身 Q_hold 通过？
6. 是否误以为同比减小同质仓位能把负期望变正，或机械移动到 breakeven、放宽 outer stop、拉远目标、未授权加仓？
7. 原计划终止后，是否先以 A3 让其仓位归零，再从新的 K / C / T / D_init / Q_entry 开始，而不是机械改名、过户仓位或直接反手？

## 最终归类

复盘结束只保留一种主要结论：

- **合格执行**：当时的 K、C、T、D_init、Q_entry 和权限完整；成交后按 D_live、E、Q_hold、A 执行。盈亏不改变该分类。
- **合格但未成交**：入场闸门合格，订单按规则等待并最终撤销、到期或未成交；P0 / A0，但不是设计错误。
- **有效候选，执行错误**：命题可成立，但订单边界、D_init、仓位、stop、target、D_live 或管理被破坏。
- **只有 pattern / 方向判断**：存在形态或方向观点，但没有完整可交易命题。
- **应当 P0**：无 K、C0、无 T、D_init 不完整或 Q_entry 不通过；等待才是正确动作。

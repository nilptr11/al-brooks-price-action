# 执行与复盘

> **定位：供交易者选择执行环境，并从观察一直走到复盘**
>
> 本目录不判断某个形态是否值得交易，也不另写一套策略条件。策略页面负责说明看什么、何时考虑、何时放弃以及怎样计划；本目录负责让交易者在历史回放、模拟或实盘中安全地执行同一套策略，并留下能够复核的过程。

## 先分清三个概念

- **一轮执行**：一次有明确开始和结束边界的观察与决策过程。它可以没有候选、没有订单，也没有成交。
- **一个机会**：一轮执行中自然出现、已经达到某个策略候选条件的情况。它可以继续等待，也可以被拒绝、失效、过期、未成交或成交。
- **一笔交易**：一个机会已经按照本轮采用的成交依据产生了实际或模拟成交。

每轮执行都要记录和复盘。没有出现机会，不代表这一轮没有价值；出现了形态，也不等于已经形成机会；价格触发了入场条件，更不等于已经成交。

## 所有环境共用的流程

1. 选择历史回放、模拟或实盘，并写清本轮开始位置、结束条件和个人风险边界。
2. 限定本轮允许使用的策略，确认自己知道每个策略的候选、放弃和计划要求。
3. 只使用当时已经出现的信息连续观察，并按所选环境要求追加记录。
4. 达到策略的候选条件时，为这个机会建立机会页；普通形态继续留在观察时间线中。
5. 逐项判断背景、位置、结构、目标空间和风险。看不清就等待，不合格就拒绝。
6. 机会合格后，在采取入场动作前一次写清入场、数量、止损、目标、取消、失效和持仓管理方法，并锁定计划。
7. 只依据所选环境能够确认的事实更新订单、成交、仓位和保护。价格到达、发出请求、收到确认和实际成交必须分开。
8. 订单与仓位全部结束后核对事实；如果只能做阶段性截点，就明确保留尚未结束的订单、仓位、保护和问题，恢复时继续使用同一轮记录与原机会页。
9. 最后分别复盘市场读取、策略遵守、计划质量、执行安全和市场结果，再写下下一次最需要改善的一步。

## 三种环境不改变策略

历史回放、模拟和实盘使用相同的策略条件、相同的放弃标准和相同的复盘问题。不同的只是：未来是否可见、成交由什么证据确认、平台安全检查做到什么程度，以及结果承担什么后果。

一轮执行开始后不得中途切换环境。历史回放不能因为后来看到结果而改成“当时会成交”；模拟结果不能改写成实盘成交；实盘中的平台事实也不能用图表推测代替。

每次执行都会产生复盘反馈，但实盘不是试验未确认规则的地方。本仓库不设定具体风险数值，不证明策略盈利，也不授权任何真实资金交易。
